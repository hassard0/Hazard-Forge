# Slice GI5 — Deterministic Lumen-class GI: INTEGER CHEBYSHEV OCCLUSION WEIGHTING (leak-fix) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (+ Metal). The FIFTH slice of FLAGSHIP #29
> (DETERMINISTIC LUMEN-CLASS GI, `engine/render/gi.h`, namespace `hf::render::gi`, slice-tag **GI**). GI1–GI4 built
> the probe trace, integer SH (the crux), the trilinear field, and multi-bounce. GI5 fixes the classic probe-volume
> artifact — **light leak**: the unoccluded trilinear blend (GI3) lets a probe on the far side of a wall contribute

> free) and weight each of the 8 trilinear corners by the probability the query point is visible to that probe. All
> integer → byte-exact, strict-zero. APPEND to `gi.h` (GI1–GI4 BYTE-FROZEN). Branch: `slice-gi5`. See
> [[hazard-forge-gi29-roadmap]], [[hazard-forge-gi-roadmap]], [[hazard-forge-docs-style]],

## Design call: distance-moment Chebyshev — the moments come free from the RT trace
The float DDGI needed a whole separate distance-cube render pass for occlusion; here the **RT probe trace already
returns the hit distance `t`** (`rtrace::TraceClosest` → `RtHit.t`), so the per-probe moments are a cheap by-product
of the GI1-style trace — a clean win. For each probe: `meanDist = avg(t over 16 rays)`, `meanDist2 = avg(t² )`
(int64 accumulate, quantize once — the GI2 discipline). Visibility of a query point at distance `d` from probe `p`:
- `if d ≤ meanDist: vis = kOne` (in front of / at the probe's average occluder — fully visible).
- `else: variance = meanDist2 - meanDist²; vis = variance / (variance + (d - meanDist)²)` (the Chebyshev upper bound
  on `P(dist ≥ d)`), clamped to `[0,kOne]`. The divide is `fxdiv` (int64). A small variance bias avoids div-by-zero.
The occlusion-weighted blend: each corner weight `w_c *= lerp(kOne, vis_c, occStrength)`, then RE-NORMALIZE so the 8
weights still sum to `kOne` (partition of unity preserved — no energy gain/loss). `occStrength=0` → every factor is
`kOne` → byte-identical to GI3 (the no-op). All integer → strict-zero.

## Design decisions (locked)
1. **APPEND to `gi.h`** (GI1–GI4 byte-frozen):
   - `struct FxProbeMoments { fx meanDist, meanDist2; }`. `inline void FxProbeMoments_All(const GiProbeGrid& grid,
     const rtrace::RtScene& scene, std::span<FxProbeMoments> out)` — per probe, trace the 16 `kGiProbeDirs` via
     `rtrace::TraceClosest`, accumulate `t` and `t²` (int64), quantize the means once. (A miss → a large sentinel
     distance — fully visible.)
   - `inline fx FxChebyshevVisibility(const FxProbeMoments& m, fx queryDist)` — the integer VSM inequality (with a
     small variance bias constant); returns `[0,kOne]`.
   - `inline GiRadiance FxInterpolateIrradianceOcc(const GiProbeGrid& grid, std::span<const FxProbeSH> shBuffer,
     std::span<const FxProbeMoments> moments, const FxVec3& point, const FxVec3& normal, fx occStrength)` — the GI3
     `FxNearestProbes` corners, each weight `*= lerp(kOne, FxChebyshevVisibility(moments[corner], distToCorner),
     occStrength)`, RE-NORMALIZE to `Σ==kOne`, then `FxInterpolateSH` + `FxSHEvaluate`. `occStrength==0` → byte-
     identical to `FxInterpolateIrradiance`.
   - `InterpolateFieldOcc(...)` + `GiOccFieldToImage(...)` (the field viz with leak-fix).
2. **NEW `shaders/gi_occ.comp.hlsl`** (one thread per query point): reads `gSH` (GI2) + `gMoments` + the query
   points, runs `FxInterpolateIrradianceOcc` VERBATIM, writes `GiRadiance`. The `fxdiv` Chebyshev → int64 → likely
   `:cs` Vulkan-only (NOT in `hf_gen_msl`, Metal CPU ref — the GI3 convention); if int32 fits, MSL-native. Document.
   The moments buffer is computed by the CPU `FxProbeMoments_All` (cheap) and uploaded (or a small GPU moments pass
   — implementer's choice; the HW==CPU proof is on the occlusion-weighted interp).
3. **Showcase `--gi5-occlusion-shot <out>` (Vulkan) AND `--gi5-occlusion` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--gi5-occlusion` BEFORE reporting DONE).** A scene with an OCCLUDER WALL between a lit
   region and a shadowed region (so the leak is visible + the fix measurable). Build grid + SH (GI1+GI2) + moments;

## Out of scope (YAGNI)
The lit GI capstone (GI6). A separate distance-cube render pass (the RT trace `t` gives the moments free). Per-ray
visibility / cone tracing (the Chebyshev statistical bound, like DDGI). Backface/normal-based weighting (a possible
refinement; GI5 is distance-based Chebyshev only). A float occlusion variant (strict integer). GI5 claims ONLY: a
deterministic integer Chebyshev occlusion weighting that reduces probe-volume light leak, byte-exact GPU==CPU, with
the no-op + leak-attenuation guards, strict-zero cross-vendor.
