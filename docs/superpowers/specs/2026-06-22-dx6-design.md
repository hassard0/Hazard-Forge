## Design call: sweep every rollback point over the DX5 canonical scene; the report is the PASS matrix; the fuzzer "fires" via the diverged flag
DX5 built the canonical replay scene + the per-point oracle is already in `RunVerdictRollback` (the
`outCorrectedEqAuthority` + `outMispredictDiverged` out-params). DX6 just sweeps it:
- **`RunDeterminismStress` loops `rollbackAt` from 0 to `ticks` inclusive** (`ticks+1` points), each calling the frozen
  `RunVerdictRollback`. For a deterministic engine, EVERY point's `corrected` is true: no matter where you snapshot +
  mispredict + rollback, you recover the authority bit-exactly. `allCorrected == points` is the headline.
- **The `diverged` flag makes the fuzzer non-vacuous** — it records whether the injected misprediction actually
  changed the world at that point. With a REAL mispredict stream (differs from auth), early points diverge (the
  misprediction has ticks to take effect); the last point (`rollbackAt == ticks`, zero speculative ticks) cannot
  diverge (nothing is mispredicted). So `allDiverged < points` is EXPECTED and correct — the report records the exact
  per-point divergence pattern, which is itself deterministic and golden.
- **The "fuzzer fires" proof is the meaningful-flag control:** run the stress with `mispredictStream == authStream` (NO
  perturbation) → `diverged` is false at EVERY point (the harness correctly reports no divergence when there is none);
  run with the real (perturbed) mispredict → `diverged` is true at the points where the misprediction takes effect.
  The flag responds to real divergence — it isn't always-true. AND `corrected` stays true in BOTH cases (the rollback
  always recovers). This is the regression value: if a determinism bug were introduced, some point's `corrected` would
  flip false and the report bytes would change.
- **The golden `stress_report.json`** is the full per-point matrix `{points, allCorrected, allDiverged, perPoint:[{rollbackAt,
  corrected, diverged}]}` — deterministic text, byte-compared. A determinism regression anywhere in the rollback
  machinery moves a `corrected` bit and breaks the golden.

## Out of scope (YAGNI)
Random/seeded tick REORDERING beyond the snapshot-point sweep (a real engine has a deterministic single tick order;
reordering would itself be a non-determinism, out of contract). Multi-threaded / cross-process stress. Fuzzing the
command CONTENT space (DX6 sweeps rollback POINTS over a fixed stream — content fuzzing is a future extension). DX6
claims ONLY: a deterministic sweep proving every rollback/snapshot point recovers the authority bit-exactly, with a
golden PASS matrix and a meaningful divergence flag — the moat stress-tested and golden-locked.
