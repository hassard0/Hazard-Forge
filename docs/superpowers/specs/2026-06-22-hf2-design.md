## Design call: the box warm friction solver, hull inertia substituted — the WH3→HF2 mirror
`persist::SolveFrictionWarm` is the accumulated, warm-started, cone-clamped tangent solver, ALREADY proven bit-exact
for box contacts (the friction flagship). Its body is: prime (seed the accumulated impulses from the cache once), the
accumulated normal block (the relative normal velocity → a Lagrange `dλ`, the accumulated `λ` clamped ≥ 0, apply
`dλ·n`), then the two tangent blocks (relative tangent velocity along `t1`/`t2` → `dλt`, the accumulated `λt` clamped
to `[-μ·λn, +μ·λn]` — the friction cone on the CURRENT accumulated normal — apply `dλt·t`). HF2 reuses this body
*verbatim*, swapping the box inverse-inertia for the hull's `manifold::WorldInvInertiaFull` tensor — EXACTLY how
`warmhull::SolveHullManifoldWarm` already adapted the normal-only warm solver to hulls. The tangent frame is HF1's
per-pair basis; the accumulators are HF1's `HullFrictionManifold.pts[].tangentImpulse1/2`; the warm seed comes from
HF1's `MatchHullFrictionCache` (which already cold-starts the tangents on a basis-axis flip — the crux is handled by
HF1's cache logic, consumed here).
- **The warm-start lever (the proof):** with the cache primed (warm), the solve reaches a LOWER residual at the same
  iteration count than a cold solve (impulses reset) — the whole point of warm-starting. HF2 measures both and
  asserts `warmResidual < coldResidual`.
- **All integer:** the Lagrange divides use `fxdiv` (int64); the cone clamp is integer min/max; GPU==CPU bit-exact.
  No HF2 step yet (HF3) — HF2 solves a FIXED manifold set in place (the contacts are given, not re-integrated).

## Design decisions (locked)
1. **APPEND to `hullfric.h`** (HF1 byte-frozen):
   - `struct HullFrictionConfig { fx mu; fx restitution; int iters; fx slop; fx beta; }` (the cone coefficient + the
     solve knobs).
   - `inline void SolveHullFrictionWarm(fpx::FxBody& a, fpx::FxBody& b, HullFrictionManifold& m, const FxMat3&
     invIaW, const FxMat3& invIbW, const HullFrictionConfig& cfg)` — the `persist::SolveFrictionWarm` body with the
     hull inertia: prime from `m.pts[].normalImpulse/tangentImpulse1/2`, `cfg.iters` Gauss-Seidel sweeps each doing
     the normal block + the two cone-clamped tangent blocks per contact, write the accumulated impulses back into
     `m.pts[]`. PINNED point order + the `manifold::WorldInvInertiaFull` lever-arm math (mirror `SolveHullManifoldWarm`).
   - `inline fx HullContactResidual(const HullFrictionManifold& m, ...)` — the post-solve constraint residual
     (max |relative normal velocity that should be ≥0 but is <0| + |tangent velocity beyond the cone|) for the
     warm<cold proof.
   - `inline void StepHullFrictionSolveOnly(world, cache, cfg, ...)` — build the manifolds (HF1) + warm-seed + run
     `SolveHullFrictionWarm` per pair in the FIXED i<j order, writing back the cache (the HF3 step minus the
     integrate/de-pen — HF2 is the solve in isolation).

   convention; chunked 1-iteration/dispatch if needed for TDR — see [[hazard-forge-gpu-tdr-chunking]]). Writes the
   solved bodies + the manifold accumulators. DXC `:cs` in `samples/hello_triangle/CMakeLists.txt` ONLY; the
