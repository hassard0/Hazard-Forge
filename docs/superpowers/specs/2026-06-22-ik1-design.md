# Slice IK1 — Deterministic IK Control-Rig: THE FIXED-POINT ANGLE LUT (the beachhead) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (RTX 4070) + Mac+Metal. The FIRST slice of
> FLAGSHIP #32 (DETERMINISTIC IK CONTROL-RIG, slice-tag **IK**). The flagship extends the determinism moat into
> character animation — bit-identical cross-platform lockstep-replayable IK. The make-or-break feasibility crux is
> that the engine has NO fixed-point `acos`/`atan2`/`sin`/`cos` (only `fpx::FxISqrt`), and IK's law-of-cosines limb
> solve needs them. IK1 RETIRES that risk in isolation, before any solver depends on it: a host-snapped Q16.16 angle
> LUT (`acos` / `atan2` / `sin` / `cos`) computed in `double` at build time and looked up with PURE INTEGER ops on the
> hot path — zero runtime transcendentals, bit-identical CPU/Vulkan/Metal. This is the `fluid.h::BuildKernelTable`
> precedent generalized from per-scene constants to a swept table, and the same doctrine `joint.h` already follows for
> its cone limit (host `cos(θ/2)` constants). **NEW header `engine/anim/ik.h`** (`#include`s `anim/skeleton.h` +
> `anim/animation.h` + `sim/fpx.h` + `sim/joint.h` READ-ONLY, all byte-frozen). Branch: `slice-ik1`. See
> [[hazard-forge-ik-roadmap]], [[hazard-forge-docs-style]], [[hazard-forge-build-env-vcvars]],
> [[hazard-forge-metal-showcase-gate]], [[hazard-forge-dx-metal-handler-compile-gate]].

**Goal:** Create `engine/anim/ik.h` (`hf::anim::ik`): a host-baked Q16.16 angle LUT + the integer lookups —
`FxAcosLut(x)` (x a Q16.16 cosine in `[-kOne, kOne]` → angle in Q16.16 radians `[0, π]`), `FxAtan2Lut(y, x)`,
`FxSinLut(θ)`, `FxCosLut(θ)` — each a PURE-INTEGER bin index + fixed-point-lerp lookup (NO runtime float/transcendental).
Add `shaders/ik_angle.comp.hlsl` (pure-int32 → MSL-NATIVE, runs on BOTH backends) that evaluates the LUTs over a fixed
sweep and is memcmp'd GPU==CPU. Add `--ik1-angle-shot` (Vulkan) / `--ik1-angle` (Metal) rendering a pure-integer LUT
visualization. Bake the **strict-zero** golden `ik1_angle`. No solver yet — this is the transcendental primitive every
later IK slice indexes.

## Design call: host-snapped LUT + deterministic fixed-point linear interpolation; pure-int32 so it MSL-gens native
- **Build at host time, look up with integers.** `BuildAcosLut(bins)` evaluates `std::acos` in `double` at `bins`
  sample points across the cosine domain `[-1, 1]`, rounds each result (in radians) to nearest Q16.16, and stores a
  `std::array`/`std::vector<fx>` (Q16.16). The `fluid.h::BuildKernelTable` mold VERBATIM (host `double` eval →
  round-to-nearest snap → integer table). Same for `sin`/`cos` over `[0, 2π)` and `atan2` (see below). The tables are
  `constexpr`-or-static, computed ONCE.
- **The lookup is pure int32 + a deterministic fixed-point lerp between the two bracketing bins** (better accuracy than
  nearest-bin, still bit-exact): clamp the input to the domain, compute the bin index + the sub-bin fraction with
  integer arithmetic, and return `table[i] + (((table[i+1] - table[i]) * frac) >> kFrac)`. Pick `bins` so the index
  math stays within **int32** (e.g. `acos`: `bins = 256`, domain width `2*kOne = 131072`; `(x + kOne) * (bins-1)` ≤
  `131072 * 255 ≈ 33.4M` < INT32_MAX — int32-safe). int32-only is what lets `ik_angle.comp` lower to MSL and run on
  BOTH backends (the FPX2 int32 broadphase precedent), giving a REAL GPU==CPU memcmp on Metal, not a CPU-ref fallback —
  the strongest determinism tier, established first.
- **`atan2`:** a 2D table or the standard reduction (compute `atan(|y/x|)` from a `[0,1]`-domain `atan` LUT, then place
  into the correct quadrant by the signs of `y`/`x` with integer branches). Keep the division `y/x` via `fxdiv`
  (int64) ONLY if needed — prefer a formulation that stays int32 for the MSL-native tier; if `atan2` forces int64,
  split it so `acos`/`sin`/`cos` (the IK2 critical path) stay int32-native and `atan2` (IK3's look-at) may be the
  int64 part. Document the split.
- **The viz golden:** `--ik1-angle` renders a PURE-INTEGER bar chart / curve plot of the LUT outputs over the sweep
  (e.g. the `acos` curve and the `sin`/`cos` curves as integer-plotted pixel columns) — deterministic, identical on
  both backends by construction (strict zero-differing-pixel). The 2D plot is integer pixel-set (the wh*/fric* 2D
  side-view convention), NOT a 3D render.

   `hf_gen_msl` MSL-gen in `metal_headless/CMakeLists.txt` (it IS MSL-native — int32). If `atan2` needs int64, put
   ONLY the acos/sin/cos sweep in the MSL-native shader and prove atan2 GPU==CPU on the Vulkan side / CPU-ref on Metal.

## Out of scope (YAGNI)
Any IK SOLVER (IK2 two-bone, IK3 FABRIK — they index this LUT). Higher-order interpolation (linear-lerp only). A
runtime-resizable table (fixed `bins`). `asin`/`tan`/`exp` (only the four IK needs). IK1 claims ONLY: a deterministic,
bit-identical, cross-platform fixed-point `acos`/`atan2`/`sin`/`cos` via host-snapped LUT, GPU==CPU, strict-zero
cross-vendor, within a documented LSB band of the analytic angle.
