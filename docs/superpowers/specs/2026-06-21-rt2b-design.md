## Design call: Metal `intersection_query` mirrors the Vulkan `RayQuery` drain 1:1

`metal::raytracing::intersection_query` (Metal 2.4+) is the inline ray-query primitive — the exact analog of HLSL
`RayQuery`. The kernel mirrors `rt_query.comp.hlsl` line-for-line:
1. Generate the SAME integer primary ray (the frozen `PrimaryRay` fx math; one thread per pixel).
2. Build a float `ray` (`FxToFloat` of the fx ray — the ONLY float, for the BVH traversal).
3. `intersection_query<...> q; q.reset(ray, accelStructure, ...);` then `while (q.next())`: for each
   `candidate_bounding_box_primitive`, read `q.get_candidate_primitive_id()`, look up the primitive (the same
   sphere/AABB SSBO data), run OUR frozen fx `IntersectSphere`/`IntersectAabb`, fold into a running closest by the
   total order `(t, primIndex)` — **never** call `q.commit_bounding_box_intersection(...)` and never narrow the ray
   interval (drain every candidate). The HW BVH is ONLY a candidate generator; OUR fx min owns the result.
4. `ShadeHitInt` (integer) → RGBA8 → output buffer.
Because the closest-hit is OUR `(t,primIndex)` integer reduction (never the query's float t), the image is identical to
the CPU reference and the Vulkan HW image, independent of Apple's BVH order/layout. The AABB margin (C1, same `kOne/64`
as RT2) makes the float traversal a strict superset → no missed candidate.

## The accel-structure build (Metal, in visual_test.mm)
- ONE `MTLAccelerationStructureBoundingBoxGeometryDescriptor` holding ALL N bounding boxes (sphere bounds + box AABBs,
  each inflated by `kRtAabbMargin`), so `get_candidate_primitive_id()` indexes directly into a parallel primitive-info
  buffer (the same {kind, sphere/aabb fields, primIndex} the kernel reads). (Metal's per-geometry `primitive_id`
  replaces Vulkan's per-instance `instanceCustomIndex` tagging — simpler: one geometry, N boxes, primitive_id 0..N-1.)
- Build: `MTLPrimitiveAccelerationStructureDescriptor` → `device.accelerationStructureSizesWithDescriptor` → allocate
  the accel structure + a scratch buffer → `encoder.buildAccelerationStructure(...)` on a command buffer → commit +
  `waitUntilCompleted` (the synchronous build, mirroring the Vulkan `vkQueueWaitIdle` path).
- Gate on `device.supportsRaytracing` (Apple-silicon M3/M4 = true; M1/M2 = false). If false → fall back to the CPU
  `rtrace::RenderScene` (like the Vulkan `hwRayQuery:false` fallback) and print `supportsRaytracing:false → SW`. The
  bench Mac is M4 → the HW path runs.

## Design decisions (locked)
1. **NEW `shaders/rt_query.metal`** (hand-authored, the FIRST in the repo): `#include <metal_stdlib>` +
   `#include <metal_raytracing>`; `using namespace metal; using namespace metal::raytracing;`. A `[[kernel]]` with the
   primitive SSBOs (sphere/aabb arrays + params), the accel structure (`instance_acceleration_structure` or
   `primitive_acceleration_structure`), and the output RGBA8 buffer, all via `[[buffer(n)]]`. int64 via `long`. The
   `intersection_query<>` drain (NO commit). Reproduces the frozen fx math VERBATIM. Compiled at runtime with
   `MTLLanguageVersion2_4`.
2. **NEW Metal showcase `--rt2-query-hw`** in `visual_test.mm` (`RunRt2QueryHwShowcase`): build the RT2 scene (reuse
   `RunRt2QueryShowcase`'s exact construction), the accel structure, dispatch the kernel, read back, ALSO run the CPU
   `rtrace::RenderScene`, memcmp HW==CPU strict, two-run determinism, `WritePNG`. If `!supportsRaytracing` → CPU
   fallback + the `supportsRaytracing:false → SW` line. **WIRE the arg dispatch** (the `--rt2-query` precedent at the
   `main()` dispatch chain).
3. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) Metal HW == CPU:** `rt2-query-hw: {rays:<R>, prims:<P>} Metal-HW==CPU image BYTE-EQUAL (supportsRaytracing:true)`.
   - **(2) determinism:** `rt2-query-hw determinism: HW two runs BYTE-IDENTICAL`.
   - **(3) cross-path closure (controller, at bake):** the baked `rt2_hwquery.png` is byte-identical to the committed
     `rt2_query.png` (Metal-HW == Vulkan-HW == CPU — the full three-way closure across both HW vendors).
4. **Golden:** NEW `tests/golden/metal/rt2_hwquery.png` — Mac-baked by the CONTROLLER via the Metal HW path; byte-
   identical to `rt2_query.png` (a distinct golden that asserts "this image was produced by Metal hardware ray
   query"). DO NOT commit a hand-made copy — bake it on the M4. Existing 241 goldens UNTOUCHED.
5. **Introspect.** Add EXACTLY `rt2-query-hw` (features). (NO `--*-shot` showcase entry — RT2b's showcase is Metal-only
   `--rt2-query-hw`, and the introspect showcase list is the Vulkan `--*-shot` manifest; add the FEATURE only, plus a
   showcase entry IF the convention includes Metal-only flags — check `RunRt2QueryShowcase`'s introspect treatment and
   mirror it.) Update `tests/introspect_test.cpp`. CONTROLLER rebakes the JSON golden (verify it stages ` M `).
6. **Cross-backend bar:** strict **0.0000** (integer shade). `verify.ps1` Mac-renders `--rt2-query-hw` via Metal HW and
   compares to the committed `rt2_hwquery` golden at 0.0000 (the per-backend determinism gate). The controller confirms
   `rt2_hwquery.png == rt2_query.png` byte-for-byte at bake (the cross-vendor/cross-path closure).
7. **No engine RHI / no Vulkan change.** The whole slice = `shaders/rt_query.metal` (NEW) + `visual_test.mm`
   (additive showcase) + `metal_headless/CMakeLists.txt` (copy/install the `.metal` so the headless target finds it at
   runtime — mirror how the `.spv`/`.gen.metal` are staged) + `verify.ps1` (1 golden) + `introspect.cpp` +
   `introspect_test.cpp`. `engine/rhi_metal/**`, `engine/rhi_vulkan/**`, `rtrace.h`, `rt_query.comp.hlsl`,
   `rt_trace.comp.hlsl`, ALL sim files, and the existing 241 goldens are BYTE-UNCHANGED.

## Out of scope (YAGNI)
RT shadows / AO (RT3). RT reflections (RT4). Triangle geometry / float radiance (Tier B / RT6). A deterministic accel
build (we depend ONLY on candidate completeness over the inflated boxes). Routing Metal HW ray query through the engine
`rhi_metal` RHI seam (RT2b proves the capability in the showcase harness; a productized Metal RT RHI is a later
concern). M1/M2 HW ray tracing (those lack it → CPU fallback; the bench M4 carries the HW proof). RT2b claims ONLY: on
Apple-silicon HW (M4), an inline `intersection_query` closest-hit byte-identical to the frozen SW reference and to the
Vulkan HW image — deterministic HW ray tracing on a SECOND vendor, closing "HW RT across Vulkan AND Metal."
