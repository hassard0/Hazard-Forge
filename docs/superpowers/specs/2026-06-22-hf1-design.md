# Slice HF1 — Hull Friction + Joints: TAGGED FRICTION MANIFOLD ON THE EPA NORMAL (beachhead) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (RTX 4070). The FIRST slice of FLAGSHIP #30
> (HULL FRICTION + HULL JOINTS — finishing the deterministic convex-rigid story, slice-tag **HF**). The general
> convex-hull world (`gjk::HullWorld` / `warmhull`) has warm-started, sleeping, robustly-stacking contacts — but they
> are FRICTIONLESS (normal-only), and the bodies cannot be JOINTED. Flagship #30 brings the two capabilities that
> already exist *for boxes* (`fric.h` tangential friction, `joint.h` ball/hinge/limit joints) to general convex
> hulls. HF1 is the friction beachhead: the GJK/EPA keyed hull manifold gains a per-contact **tangent basis** (built
> once per pair on the sign-corrected EPA normal) + the tangent-impulse accumulator fields + the warm-start cache
> fields — the data the HF2 cone solver consumes. **NEW header `engine/sim/hullfric.h`** (`#include`s
> `sim/warmhull.h` + `sim/gjk.h` + `sim/fric.h` + `sim/persist.h` READ-ONLY — warmhull.h is BYTE-FROZEN, NOT
> modified). Branch: `slice-hf1`. See [[hazard-forge-warmhull-roadmap]], [[hazard-forge-strategic-roadmap]],
> [[hazard-forge-docs-style]], [[hazard-forge-gpu-tdr-chunking]], [[hazard-forge-introspect-golden-staleness]],
> [[hazard-forge-metal-showcase-gate]].

**Goal:** Create `engine/sim/hullfric.h` (`hf::sim::hullfric`): a `HullFrictionManifold` (the warmhull keyed-manifold
contact points/normal/depth + per-point integer tangent basis `t1`/`t2` + zeroed tangent-impulse accumulators), a
`CachedHullFrictionContact` (the `HullContactKey` + `normalImpulse` + `tangentImpulse1`/`2` + the **tangent-basis-axis
index** — the crux field that makes the HF2 warm-start safe across a basis flip) + `MatchHullFrictionCache`/
`UpdateHullFrictionCache`, and `BuildHullFrictionManifold` = the FROZEN warmhull/gjk manifold builder (read-only) +
`fric::MakeTangentBasis(sign-corrected EPA normal)` ONCE per pair. Add `shaders/hullfric_points.comp.hlsl` (one
thread per hull pair; int64 → Vulkan-only). Add `--hf1-points-shot` (Vulkan) / `--hf1-points` (Metal); prove the
friction manifold is **bit-exact GPU==CPU**, the tangent basis is orthonormal on every EPA normal, and the contact
positions are byte-equal to the warmhull manifold. Bake the **strict-zero** golden `hf1_points`.

## Design call: a NEW friction header that wraps the FROZEN warmhull manifold; basis on the EPA normal
`warmhull.h` is a completed flagship's header — byte-frozen, `#include`d read-only by `gjk`/`ccd`/etc. HF1 must NOT
grow `warmhull::KeyedHullManifoldWH2` / `CachedHullContact` in place (that changes byte layout → breaks the `wh*`
goldens). Instead `hullfric.h` defines NEW structs that WRAP the warmhull data + add the friction fields, and a NEW
`BuildHullFrictionManifold` that CALLS the frozen warmhull/gjk manifold builder read-only and appends the tangent
basis. The friction itself is the `fric.h`/`persist.h` machinery, applied to the hull manifold:
- **The tangent basis is built on the CONTINUOUS EPA normal**, not a SAT axis. `fric::MakeTangentBasis(n)` (integer
  Gram-Schmidt from a unit normal) is fed the SAME sign-corrected A→B normal `warmhull::SolveHullManifoldWarm`
  computes — ONCE per pair (all contact points of a manifold share the basis). The basis is discontinuous in `n` (it
  flips when the least-aligned cardinal axis changes), which is the HF2 warm-start crux — HF1 lays the groundwork by
  **caching the basis-axis index** so HF2 can detect a flip and safely cold-start the tangents (a deterministic
  WH1-style safe miss, integer-clean).
- **`CachedHullFrictionContact` mirrors `persist::CachedContact`** (key + 3 impulses) PLUS the basis-axis index;
  `MatchHullFrictionCache`/`UpdateHullFrictionCache` mirror `persist::MatchCache`/`UpdateCache` (warm-seed all three
  impulses when the key matches AND the basis axis is unchanged; else cold-start the tangents).

## Design decisions (locked)
1. **NEW `engine/sim/hullfric.h`** (`hf::sim::hullfric`), the flagship's append-only header (frozen per slice;
   `#include`s warmhull.h/gjk.h/fric.h/persist.h/manifold.h READ-ONLY). Contents:
   - `struct HullFrictionPoint { FxVec3 point; fx normalImpulse; fx tangentImpulse1, tangentImpulse2; }` (per-point
     accumulators) + the manifold-level basis.
   - `struct HullFrictionManifold { /* the warmhull manifold data: */ int count; FxVec3 normal; FxVec3 points[4];
     fx depths[4]; /* + the friction basis (one per manifold): */ FxVec3 t1, t2; int32_t basisAxis; /* the
     LeastAlignedAxis index */ HullFrictionPoint pts[4]; }` — pad for the std430 GPU mirror.
   - `inline HullFrictionManifold BuildHullFrictionManifold(const gjk::FxBody& a, const gjk::FxHull& ha, const
     gjk::FxBody& b, const gjk::FxHull& hb, ...)` — call the FROZEN warmhull/gjk manifold builder (read-only) → the
     contact points + sign-corrected EPA normal; `fric::MakeTangentBasis(normal)` ONCE; `basisAxis =
     fric::LeastAlignedAxis(normal)`; zero the tangent accumulators. Pure integer.
   - `struct CachedHullFrictionContact { warmhull::HullContactKey key; fx normalImpulse, tangentImpulse1,
     tangentImpulse2; int32_t basisAxis; }` + `MatchHullFrictionCache(cache, key, basisAxis)` (mirror
     `persist::MatchCache`: warm-seed iff key matches AND basisAxis matches; else the tangents cold-start) +
     `UpdateHullFrictionCache`.
   - `BuildAllHullFrictionManifolds(world, ...)` — per i<j hull pair (the fixed-order all-pairs of `StepWarmHullWorld`).
2. **NEW `shaders/hullfric_points.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64, `:cs`): one thread per hull PAIR; runs
   `BuildHullFrictionManifold` VERBATIM (the GJK/EPA + the integer tangent basis), writes the `HullFrictionManifold`
   to an SSBO. Register a DXC `:cs` line in `samples/hello_triangle/CMakeLists.txt` ONLY; the Vulkan-only comment
   block in `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line (int64 → Metal runs the CPU
   `BuildAllHullFrictionManifolds`, byte-identical by construction — the warmhull `hull_manifold.comp` convention).
3. **Showcase `--hf1-points-shot <out>` (Vulkan) AND `--hf1-points` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--hf1-points` BEFORE reporting DONE).** A FIXED set of resting/overlapping hull pairs (the
   warmhull WH-scene's hulls, or a small curated set: a hull on a tilted hull, a few stacked). Vulkan: dispatch
   `hullfric_points.comp` → read back the manifolds; compute the CPU `BuildAllHullFrictionManifolds`; memcmp GPU==CPU
   strict; render a PURE-INTEGER XZ top-down viz (each pair's hull footprints + the per-contact tangent frame: `t1`
   cyan, `t2` magenta, `n` white — the `fric` golden style). Golden = `tests/golden/metal/hf1_points.png`
   (CONTROLLER-baked).

## Out of scope (YAGNI)
The warm cone solver (HF2), the friction-locked hull step (HF3), hull joints (HF4), lockstep (HF5), the lit capstone
(HF6). Non-canonical / non-convex hulls (the warmhull line is canonical-convex-only). Rolling/spinning/anisotropic
friction (HF is Coulomb-cone only). HF1 claims ONLY: the friction-extended keyed hull manifold — a per-pair integer
tangent basis on the sign-corrected EPA normal + the tangent-impulse accumulator + cache fields (incl. the basis-axis
index for the safe warm-start) — bit-exact GPU==CPU, the contact geometry byte-equal to the frozen warmhull manifold,
strict-zero cross-vendor. The data the HF2 cone solver consumes.
