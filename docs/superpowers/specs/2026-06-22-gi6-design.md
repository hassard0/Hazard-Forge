## Design call: compose the whole GI pipeline into the lit render; keep it strict integer
GI6 is composition, not a new technique: the GI1–GI5 functions already produce a deterministic integer irradiance
field; GI6 BAKES that field for the hero scene and SAMPLES it per primary hit, adding `albedo × indirectIrradiance ×
giStrength` to the RT6 direct+reflected shade. All integer → the GI-lit hero is byte-identical HW==CPU and strict-
zero cross-vendor. The bake (trace → SH → K-bounce → moments) runs once per render; the primary pass then does one
`FxInterpolateIrradianceOcc` lookup per hit.
- **`giStrength=0` → a literal integer +0 indirect → byte-identical to the no-GI hero** (the RT6-style render). A
  real no-op, the falsifiable guard that the GI term is purely additive.
- **The Cornell enclosure** (a floor + ceiling + back wall + a RED left wall + a GREEN right wall + a couple objects,
  a directional/overhead light) is the canonical GI scene: the colored walls catch direct light and bounce their hue
  onto the floor + objects — the multi-bounce color bleed that single-bounce direct lighting cannot produce.
- **Strict integer, NOT float** — the RT6 posture: a float Tier-B fidelity GI is a future fast-follow; the capstone
  golden stays strict-zero so the flagship is bit-identical end-to-end (the maximal moat statement).

## Design decisions (locked)
1. **APPEND to `gi.h`** (GI1–GI5 byte-frozen):
   - `inline constexpr fx kGiDefaultStrength = ...;` (the indirect gain, ≤kOne).
   - `inline uint32_t RenderSceneGI(const rtrace::RtScene& scene, const GiProbeGrid& grid, std::span<const FxProbeSH>
     sh, std::span<const FxProbeMoments> moments, const rtrace::RtCamera& cam, uint32_t w, uint32_t h, fx giStrength,
     std::span<uint32_t> out)` — per pixel: the RT6 hero shade (primary `TraceClosest` → `ShadeHitShadowed` + the
     reflection bounce + `SkyGradient` on miss), PLUS `indirect = fxmul(albedo(hit) ⊙ FxInterpolateIrradianceOcc(grid,
     sh, moments, hit.pos, hit.normal), giStrength)` added to the hit color (per channel, clamped). `giStrength=0`
     → byte-identical to the no-indirect hero. Returns a small struct/counts (e.g. indirect-lit pixel count) for the
     proof.
   - `inline GiBakeResult BakeGI(scene, grid, bounces)` (or reuse: the showcase runs `BounceProbes` + `FxProbeMoments_All`
     directly) — produces the `sh` + `moments` the render samples.
   - `BuildGi6Scene(...)` — the PINNED Cornell enclosure (red + green walls, floor/ceiling/back, objects, light).
2. **NEW `shaders/gi_hero.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq`): the `rt_hero.comp` body + the

   `samples/hello_triangle/CMakeLists.txt` ONLY; the Vulkan-only comment in `metal_headless/CMakeLists.txt`; NO
   `hf_gen_msl` line. **TDR:** the GI bake (K≤3 small probe passes) + one primary pass (≤~480×360, ~3 rays/pixel +

## Out of scope (YAGNI)
A float Tier-B fidelity GI (strict integer — the moat). Specular/glossy GI (diffuse SH; specular stays on the RT4
mirror path). Infinite bounce (K≤3). Runtime dynamic GI re-bake per frame (static bake per shot, like DDGI). Real-time
probe relocation / world-scale GI streaming. GI6 claims ONLY: a polished Cornell-enclosure hero rendered with the
full deterministic integer GI pipeline (direct + shadows + reflections + sky + occlusion-weighted multi-bounce
indirect with colored bleed), byte-identical HW==CPU on Vulkan HW, strict-zero cross-vendor — the capstone proving
the whole GI pipeline is deterministic. **Completes FLAGSHIP #29.**
