## Design call: the WH6/CX6/FC6 capstone mold — a render-only float bridge, the sim untouched
Five flagships ended this exact way (wh6_render, convex_render, fric_render, fpx_render, vd6_game): the bit-exact
integer world → a one-line `…ToRenderInstances` delegate → the existing instanced-lit pipeline → a Metal-baked float
golden. HF6 IS that pattern for the friction+joint hull world:
- **The bridge is a delegate, not new math.** `JointedHullToRenderInstances(world)` just calls
  `gjk::HullToRenderInstances(world.hulls)` — the bodies are `gjk::HullWorld` bodies, already renderable by the frozen
  bridge. The joints/limits are constraints, not geometry — they don't render; the hull BODIES (chain links, door,
  resting hull) render, and their settled poses ARE the joint+friction result. The provenance proof is
  `HullRenderMeshEqual(JointedHullToRenderInstances(w), JointedHullToRenderInstances(w))`.
- **MATTE (metallic 0, roughness 1)** — dodge the GF6/BD6/CX6/FC6 iridescence trap (the documented capstone pitfall;
  warm-amber dynamic bodies on a cool-grey static ramp/anchor, matte).
- **The float bar, not strict-zero.** The sim is bit-exact (HF1–HF5) but the final perspective raster + Lambert shade
  is FLOAT (≤~1 LSB cross-vendor). So the golden is the FLOAT visresolve bar: a per-backend Metal-baked golden,
  Metal==Metal-golden DIFF 0.0000 (deterministic two-run) + provenance + a DOCUMENTED in-band cross-vendor smoke
  (~20–60 mean Vulkan-vs-Metal — the float baseline, NOT zero). This is the ONE float slice in the whole flagship.
- **NO GPU sim step in the showcase** — the world is settled by the CPU `StepJointedHullWorldN` (the render is a normal
  draw, no compute dispatch, no TDR), exactly as HF6's siblings did. Both backends build the SAME scene + settle it
  identically (the integer sim is byte-identical cross-backend) → the render INPUT is byte-identical; only the float
  raster differs within the bar.
