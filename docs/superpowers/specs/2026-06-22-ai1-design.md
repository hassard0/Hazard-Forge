# Slice AI1 — Deterministic AI: THE BLACKBOARD + DECISION-TREE NODE GRAPH + DETERMINISTIC TICK (beachhead) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan + Mac+Metal. The FIRST slice of the flagship for
> **GitHub issue #28** (DETERMINISTIC AI — decision trees + environment queries, slice-tag **AI**). The engine ships a
> deterministic navmesh + A* but no NPC decision layer. This flagship extends the determinism moat into game AI — a
> bit-identical, cross-platform, lockstep-replayable decision layer that float engines run non-deterministically (so
> they cannot lockstep/replay AI). AI1 is the beachhead: a **flat, data-driven decision-tree node graph** + an
> **integer blackboard** + a **deterministic tick**, evaluating integer conditions and writing integer actions — no
> nav, no environment queries, no perception yet. It pins the two load-bearing determinism decisions the whole
> flagship rides on: a FLAT INDEX graph with a FIXED depth-first child order (not pointers/recursion — those are
> non-deterministic across allocators/compilers), and an integer blackboard digest (the proof currency every later
> slice reuses). **NEW header `engine/ai/ai.h`** (`hf::ai`, `#include`s `game/verdict.h` + `sim/fpx.h` READ-ONLY,
> byte-frozen). Header-only, pure CPU. Branch: `slice-ai1`. See [[hazard-forge-ai-roadmap]], [[hazard-forge-issue-burndown]],
> [[hazard-forge-docs-style]], [[hazard-forge-build-env-vcvars]], [[hazard-forge-metal-showcase-gate]].

**Goal:** Create `engine/ai/ai.h` (`hf::ai`): `struct Blackboard` (a fixed-capacity integer/Q16.16 key→value store —
deterministic, byte-stable), `struct BtNode { uint32_t kind; int child[kMaxChildren]; int childCount; int32_t param;
int bbKey; }` (a flat node), `enum NodeKind { kSelector, kSequence, kInverter, kCondLeaf, kActionLeaf }`, a
`DecisionTree` = `std::vector<BtNode>` (root at index 0, children referenced by index — NOT pointers), `enum Status {
kRunning, kSuccess, kFailure }`, and `Status TickTree(const DecisionTree& tree, Blackboard& bb)` evaluating the tree
depth-first in FIXED child order: a **selector** returns the first non-Failure child (short-circuits); a **sequence**
returns the first non-Success child; an **inverter** flips Success↔Failure; a **condition leaf** tests a blackboard
value against `param` (e.g. `bb[bbKey] >= param`); an **action leaf** writes `param` into `bb[bbKey]` and returns
Success. Add `DigestBlackboard(bb) -> uint64_t` (the FNV-1a digest, reusing verdict's FNV constants). Add
`--ai1-tree-shot` (Vulkan) / `--ai1-tree` (Metal) rendering a pure-integer node-status visualization. Bake the
**strict-zero** golden `ai1_tree`.

## Design call: flat index graph + fixed DFS order + integer blackboard = determinism by construction
- **Flat, not pointers/recursion.** The tree is a `std::vector<BtNode>`; children are integer indices (`child[]`),
  evaluated in FIXED order. This is the `nav::Poly` flat-index discipline. A pointer graph or unbounded recursion would
  be non-deterministic across allocators/compilers; an explicit index walk (an integer stack, NOT C++ recursion, if
  depth could blow the stack — but a bounded `kMaxDepth` recursion is fine and simpler — pick one, document it) is
  bit-stable.
- **Integer blackboard.** `Blackboard` is a fixed `std::array` of `int32_t`/`fx` slots (a small key space, e.g.
  `kMaxBbKeys = 16`), zero-initialized. Conditions/actions read/write by integer key. NO float, NO `std::map` (its
  iteration/allocation could vary) — a fixed array indexed by key. `DigestBlackboard` FNV-1a's the slot array in fixed
  order → the deterministic proof currency (reuse the `verdict.h` FNV constants `1469598103934665603` /
  `1099511628211`, a local helper, NOT pulling verdict's into ai.h if it's not already exposed — mirror it).
- **Fixed tick semantics.** Selector/sequence iterate `child[0..childCount-1]` in order; the FIRST that breaks the
  rule decides. Two `TickTree` runs over the same tree+blackboard are byte-identical by construction (fixed order,
  in-place integer mutation). The flagship's later slices (perception, EQS, lockstep) all ride this.
- **Docs-style:** ship GENERIC terms in the header/docs ("decision tree", "blackboard", "condition/action node") — no
  third-party-engine branding (the `nav.h`-avoids-"Recast" rule).
