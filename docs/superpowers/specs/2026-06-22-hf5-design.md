## Design call: the WH5/VD5 lockstep mold, with the friction cache IN the snapshot (the warm-state-necessary lesson)
The lockstep/rollback machinery is proven five times over (WH5, JT5, FR5, GR5, VD5). HF5 reuses the SHAPE verbatim; the
ONE HF-specific subtlety is WHAT goes in the snapshot:
- `StepJointedHullWorld` carries warm state ACROSS ticks in `JointedHullWorld.cache` (the `HullFrictionCache` —
  last tick's accumulated tangent/normal impulses keyed by contact). For two peers to converge bit-for-bit, BOTH must
  re-derive that cache identically from inputs (they do — it's a pure function of the tick sequence). For ROLLBACK to
  correct exactly, the snapshot MUST capture the cache too — restoring only the bodies and dropping the warm cache
  re-seeds friction from cold and DIVERGES. This is the VD4 "sim-third necessary" / WH5 "the TRIPLE" lesson at the
  friction layer. **Include a proof that the cache is necessary:** a restore that OMITS the `HullFrictionCache`
  diverges; including it converges.
- The joints/limits/config are CONST topology (not mutated by the step) → carried as `RunJointed*` params /
  `JointedHullParams`, NOT serialized in the snapshot (the VD5 const-topology pattern). `JointedHullStatesEqual`
  compares bodies + cache only.
- **Determinism by construction:** `SimJointedHullTick` is the frozen HF4 `StepJointedHullWorld` (fixed joint-list +
  i<j contact order, in-place integer mutation). Two peers running the same `SimJointedHullTick` over the same command
  stream are byte-identical; rollback restores the exact `JointedHullSnapshot` and re-sims. Pure CPU — NO GPU, NO TDR.

## Design decisions (locked)
1. **APPEND to `engine/sim/hulljoint.h`** (HF1–HF4 BYTE-FROZEN — do NOT edit existing lines):
   - `struct JointedHullSnapshot { int32_t tick; std::vector<gjk::FxBody> bodies; hullfric::HullFrictionCache cache; }`
     (deep copies).
   - `struct JointedHullParams { JointedHullStepConfig cfg; /* + the per-tick command/impulse source, see (2) */ }`
     — the const sim knobs bundled (the VD5 `VerdictParams` pattern).
   - `SnapshotJointedHull(world) -> JointedHullSnapshot` (deep-copy bodies + cache + tick).
   - `RestoreJointedHull(world, snap)` (memberwise restore bodies + cache + tick; joints/limits untouched = const).
   - `JointedHullStatesEqual(a, b) -> bool` (a.tick==b.tick AND bodies equal element-wise AND cache equal in FIXED
     order — provide a small `HullFrictionCacheEqual` helper if one isn't already reachable).
   - `SimJointedHullTick(world, params, tick)` = apply this tick's input (an external impulse on a chosen body — see
     (2)) then `StepJointedHullWorld`.
   - `RunJointedHullLockstep(scene, stream, ticks) -> JointedHullSnapshot` — clone scene into peer A + peer B (build
     two identical `JointedHullWorld`s from the same const init; NOT a copy if the world is non-copyable — rebuild),
     step BOTH over the same stream, assert `JointedHullStatesEqual` EACH tick, return the converged snapshot.
   - `RunJointedHullRollback(scene, stream, rollbackAt, mispredictTicks) -> bool` — advance to `rollbackAt`,
     `SnapshotJointedHull`, speculatively run `mispredictTicks` (≤3) with a WRONG stream, `RestoreJointedHull`,
     re-run the CORRECT stream; assert corrected == a never-mispredicted authority AND the mispredict diverged.
2. **The input/command stream** = a small per-tick external impulse on a chosen body (the chain's free end, the door,
   or the resting hull). Keep it integer + deterministic. The WRONG (mispredict) stream applies a DIFFERENT impulse
   so the mispredicted state diverges across the chain AND the door AND the resting hull (the heterogeneous-divergence
   proof, the VD5 shape). A trivial "no external input, settle under gravity" stream is INSUFFICIENT for the
   divergence proof — drive at least one impulse that the mispredict gets wrong.
3. **Showcase `--hf5-net-shot <out>` (Vulkan, `samples/hello_triangle/main.cpp`) AND `--hf5-net` (Metal,
   `metal_headless/visual_test.mm`) — WIRE BOTH (grep your OWN `visual_test.mm` for `--hf5-net` BEFORE reporting
   DONE).** Both build the IDENTICAL scene (the HF4 chain + door + resting hull) + the SAME command stream, run
   `RunJointedHullLockstep`, and render the converged authority world (the HF4-style pure-integer 2D side-view) →
   bit-identical cross-backend BY CONSTRUCTION (strict zero-differing-pixel). Golden =
   `tests/golden/metal/hf5_net.png` (CONTROLLER-baked).
