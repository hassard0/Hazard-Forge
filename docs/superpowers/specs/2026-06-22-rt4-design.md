# Slice RT4 — Hardware Ray Tracing: DETERMINISTIC RT MIRROR REFLECTIONS (strict integer) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (NVIDIA RTX 4070, HW ray query). The FOURTH slice
> of FLAGSHIP #28 (HARDWARE RAY TRACING, `hf::render::rtrace`, slice-tag **RT**). RT1 SW tracer + accel seam; RT2/RT2b
> HW ray query (Vulkan + Metal, byte-identical); RT3 deterministic hard shadows. RT4 adds **one-bounce mirror
> reflections**: a reflective surface casts a REFLECTION ray (the view dir reflected about the fx surface normal),
> the reflected hit is shaded (reusing RT3's shadowed integer shade), and the result is composited into the surface
> color by an integer reflectivity. **STRICT INTEGER** end-to-end (fx reflect + the same fx intersection/min + the
> RT3 integer shadowed shade + fx composite), so — unlike a float reflection engine — RT4 stays **strict-zero
> cross-vendor** (HW==SW==CPU bit-exact), the moat extended to reflections. The reflection ray reuses the SAME accel
> structure + the SAME candidate-drain contract. APPEND to `rtrace.h` (RT1+RT3 code BYTE-FROZEN). Branch:
> `slice-rt4`. See [[hazard-forge-raytracing-roadmap]], [[hazard-forge-docs-style]], [[hazard-forge-gpu-tdr-chunking]],
> [[hazard-forge-introspect-golden-staleness]], [[hazard-forge-metal-showcase-gate]].

**Goal:** APPEND to `engine/render/rtrace.h`: `FxReflect` (fx mirror reflect), `ReflectivityFor` (per-prim integer
reflectivity), `RenderSceneReflected` (primary shadowed shade + a one-bounce reflection composite). Add
`shaders/rt_reflect.comp.hlsl` (Vulkan inline RayQuery: primary closest-hit + primary shadow + reflection closest-hit
+ reflected shadow + integer composite). Add `--rt4-reflect-shot` (Vulkan) / `--rt4-reflect` (Metal) over an RT2-style
scene with a **reflective ground** (the floor mirrors the spheres) + a **mirror sphere**. Prove the HW reflected image
is **byte-identical** to the CPU `rtrace::RenderSceneReflected`. Bake the **strict-zero** golden `rt4_reflect`.

## Design call: one-bounce mirror reflection, fully integer → strict-zero
The reflection ray's closest hit is the SAME deterministic `(t,primIndex)` min over the accel structure (HW candidate
drain == SW brute-force == CPU). The reflected hit is shaded with the RT3 **integer shadowed** shade
(`ShadeHitShadowed` + its own any-hit shadow ray), and the composite `final = surface*(kOne-refl) + reflected*refl`
is a per-channel fx blend — all integer, so the whole reflected image is bit-exact cross-backend. No float anywhere
on the path. (This is STRONGER than the roadmap's tentative "Tier-B float composite" — integer one-bounce reflections
keep the strict-zero moat; a float multi-bounce path-traced tier is deferred to the GI flagship #29.)
- **The fx reflect:** `r = d - 2*(d·n)*n` (n unit; d the incoming ray dir, need not be unit — |r|=|d| so the
  reflected ray's `t` units are consistent). `FxReflect(d,n) = FxSub(d, FxScale(n, 2*FxDot(d,n)))`. Reflection-ray
  origin = `hit.pos + n*kRtReflEps` (anti-acne, same idiom as the shadow ray).
- **Reflectivity** is a fixed per-`primIndex` integer factor in [0,kOne]: the ground (primIndex 0) and ONE mirror
  sphere are reflective (e.g. `kOne*55/100` ground, `kOne*75/100` mirror sphere); all others 0 (matte → byte-
  identical to the RT3 shadowed shade, so non-reflective pixels are unchanged from RT3's image). One bounce only: the
  reflected hit's OWN reflectivity is ignored (no recursion) — the reflected surface is shaded matte+shadowed.
- **Reflection miss → the background** color (the reflected sky).

## Design decisions (locked)
1. **APPEND to `rtrace.h`** (RT1+RT3 byte-frozen):
   - `inline constexpr fx kRtReflEps = kOne / 256;`
   - `inline FxVec3 FxReflect(const FxVec3& d, const FxVec3& n)` — `d - 2*(d·n)*n` in fx.
   - `inline fx ReflectivityFor(uint32_t primIndex)` — the fixed per-prim factor (ground primIndex 0 → `kOne*55/100`;
     the designated mirror-sphere primIndex → `kOne*75/100`; else 0). **Pin the mirror-sphere primIndex as a named
     constant** so the scene + shader agree.
   - `inline uint32_t RenderSceneReflected(scene, cam, w, h, out)` — per pixel: primary `TraceClosest`; the primary
     shadowed color `cS = ShadeHitShadowed(hit, scene, TraceAnyHit(primaryShadowRay))`; if `hit` is reflective
     (`refl = ReflectivityFor(hit.primIndex) > 0`): reflect ray `{hit.pos + n*kRtReflEps, FxReflect(primaryDir, n)}`,
     `rHit = TraceClosest(reflRay)`, reflected color `cR = (rHit miss ? background : ShadeHitShadowed(rHit, scene,
     TraceAnyHit(reflShadowRay)))`; `out = blend(cS, cR, refl)` per channel (`c = (cS*(kOne-refl) + cR*refl) >>
     kFrac`, on the unpacked 0..255 channels). Else `out = cS`. Returns the count of REFLECTIVE pixels (a primary hit
     whose surface is reflective — the reflections-present proof).
     - **The blend operates on UNPACKED RGBA8 channels** (unpack cS/cR, blend each channel in fx, repack) — pin this
       exactly so the shader matches byte-for-byte.
2. **NEW `shaders/rt_reflect.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq`): one thread per pixel. The
   RT3 primary+shadow body, THEN if the primary hit is reflective: a reflection `RayQuery` (closest-hit drain), a
   reflected shadow `RayQuery` (any-hit), the integer blend. Copy the fx math VERBATIM from `rtrace.h`/`rt_shadow.comp`.
   DXC `csrq` line in `samples/hello_triangle/CMakeLists.txt` ONLY; Vulkan-only comment block in
   `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line.
3. **Showcase `--rt4-reflect-shot <out>` (Vulkan) AND `--rt4-reflect` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--rt4-reflect` BEFORE reporting DONE).** Scene: the RT2 scene but with the ground +
   one designated sphere reflective (so the floor mirrors the spheres and the mirror sphere reflects its
   neighbours). Vulkan (HW): build the accel structure, dispatch `rt_reflect.comp`, read back; CPU
   `RenderSceneReflected`; memcmp HW==CPU strict; write the HW image. No-RT → CPU fallback + `hwRayQuery:false → SW`.
   Metal: CPU `RenderSceneReflected`. Golden = `tests/golden/metal/rt4_reflect.png` (CONTROLLER-baked).
4. **PROOFS (fail loudly; exact stdout lines):**

## Out of scope (YAGNI)
Multi-bounce / recursive reflections (ONE bounce; the reflected surface is matte+shadowed, its own reflectivity
ignored — recursion + multi-bounce path tracing is the GI flagship #29). Glossy/rough reflections (mirror only — no
BRDF lobe sampling). Fresnel (a fixed per-prim reflectivity, not view-angle-dependent). Refraction. Metal HARDWARE
reflections (Metal runs the CPU reference; the Metal-HW reflection parity is a follow-on — RT2b proved the Metal HW
primary path). Float radiance (RT4 is strict integer — the Tier-B float path is a GI-flagship concern). RT4 claims
ONLY: deterministic one-bounce MIRROR reflections, byte-identical HW==CPU on Vulkan HW, strict-zero cross-vendor,
with a reflections-actually-contributed sanity.
