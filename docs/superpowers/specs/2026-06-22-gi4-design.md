# Slice GI4 — Deterministic Lumen-class GI: INTEGER MULTI-BOUNCE FEEDBACK — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (RTX 4070, HW ray query). The FOURTH slice of
> FLAGSHIP #29 (DETERMINISTIC LUMEN-CLASS GI, `engine/render/gi.h`, namespace `hf::render::gi`, slice-tag **GI**).
> GI1 traced probe radiance, GI2 encoded integer SH (the crux — proven in-range), GI3 gave the continuous integer
> irradiance field. GI4 closes the loop: **multi-bounce**. Each probe-ray hit's shade gains an INDIRECT term = the
> PREVIOUS iteration's GI3-interpolated irradiance at the hit point × the hit's albedo, fed back for a fixed K≤3
> iterations — the deterministic integer twin of the float DDGI `probe_multibounce`. This is the GI step that makes
> light *bounce* (a red wall tints the floor); it is the second make-or-break beat (after GI2's crux), guarded by two
> falsifiable proofs: **bounceCount=1 is byte-identical to single-bounce** (no-op) and **2nd-bounce monotonicity
> SH₂ ≥ SH₁** (the bounce adds light AND did not underflow to zero). APPEND to `gi.h` (GI1+GI2+GI3 BYTE-FROZEN).
> Branch: `slice-gi4`. See [[hazard-forge-gi29-roadmap]], [[hazard-forge-gi-roadmap]], [[hazard-forge-docs-style]],
> [[hazard-forge-gpu-tdr-chunking]], [[hazard-forge-introspect-golden-staleness]], [[hazard-forge-metal-showcase-gate]].

**Goal:** APPEND to `engine/render/gi.h`: `ShadeHitGI` (the direct shadowed shade + an indirect `albedo × interpolated

**strict-zero** golden `gi4_bounce`.

## Design call: bounded-gain integer feedback, fixed iterations — converges within the proven SH headroom
The bounce energy at iteration k is `Σ albedo^k`. Because every albedo is `< kOne` (energy-conserving) and K is
fixed (≤3), the series is bounded — `< radiance/(1-maxAlbedo)`, which GI2 proved stays within the `[-2,2]` integer-SH
headroom. So the feedback CANNOT overflow the fx bound, and the GI2 dynamic-range guarantee carries through. The
ONLY new ingredient is the indirect shade:
- **`ShadeHitGI(hit, scene, occluded, indirectIrr)`** = `albedo(hit) × (directDiffuse + indirectIrr)` where
  `directDiffuse = ambient + (occluded ? 0 : lambert)` (the `ShadeHitShadowed` body) and `indirectIrr` is the GI3
  `FxInterpolateIrradiance` of the PREVIOUS SH at `hit.pos` for the hit normal. Pure integer; when `indirectIrr == 0`
  (iteration 1, no prior SH) it is **byte-identical to `ShadeHitShadowed`** (the no-op contract).
- **`BounceProbes(scene, grid, K)`:** `SH_0` is undefined-zero; iteration 1 = `TraceProbeRays` (direct, `indirectIrr=0`)
  → `EncodeAllProbes` → `SH_1` (== GI1+GI2 single-bounce); iteration k>1 = re-trace with `ShadeHitGI(...,
  FxInterpolateIrradiance(SH_{k-1}, hit))` → `EncodeAllProbes` → `SH_k`. Return `SH_K`. Fixed K (clamp ≤3).
- **The make-or-break guards:** (a) `K==1` → `SH_1` is byte-identical to the GI1+GI2 single-bounce SH (the indirect
  term is a literal integer +0 — a real no-op). (b) `SH_2`'s reconstructed irradiance ≥ `SH_1`'s component-wise
  (the 2nd bounce only ADDS light) AND is measurably greater for ≥1 probe (so the integer indirect did NOT truncate
  to zero — the underflow guard, the GI2 crux re-verified under feedback).

## Design decisions (locked)
1. **APPEND to `gi.h`** (GI1+GI2+GI3 byte-frozen):
   - `inline constexpr int kGiMaxBounces = 3;` + `inline int ClampBounces(int k)` (1..kGiMaxBounces).
   - `inline GiRadiance ShadeHitGI(const rtrace::RtHit& hit, const rtrace::RtScene& scene, bool occluded,
     const GiRadiance& indirectIrr)` — `albedo × (directDiffuse + indirectIrr)`; `indirectIrr==0` → ==
     `ShadeHitShadowed`. (Define the per-channel integer blend explicitly; PIN it for the shader.)
   - `inline void BounceProbes(const rtrace::RtScene& scene, const GiProbeGrid& grid, int bounces,
     std::span<FxProbeSH> outSH)` — the fixed-K iteration (uses scratch radiance + the GI1/GI2/GI3 functions). Pure
     CPU reference. Returns `SH_K`.
   - (Optional) `inline GiRadiance BounceIrradianceAt(...)` helper for the proofs.
2. **NEW `shaders/gi_bounce.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq`): one thread per probe; the
   GI1 trace body, but each hit's shade adds the indirect term — the GI3 `FxInterpolateIrradiance` of the bound
   `gPrevSH` buffer at the hit point (the trilinear blend + `FxSHEvaluate`, copied from `gi_interp`/`gi.h`). Writes
   the new `GiRadiance`. The host drives the K-iteration loop (dispatch `gi_bounce.comp` reading `SH_{k-1}`, then
   `gi_sh_encode.comp` to make `SH_k`). `csrq` DXC line in `samples/hello_triangle/CMakeLists.txt` ONLY; the
   Vulkan-only comment in `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line. **TDR:** K≤3 × (64 probes × 16 rays
   × ~2 rays) — trivially under the watchdog; per-iteration dispatch (the loop is host-driven), no chunking.
3. **Showcase `--gi4-bounce-shot <out>` (Vulkan) AND `--gi4-bounce` (Metal) — WIRE BOTH (grep your OWN

## Out of scope (YAGNI)
Occlusion-weighted interpolation (GI5 — GI4's indirect uses the unoccluded GI3 blend; the leak-fix is GI5). The lit GI
capstone (GI6). Infinite-bounce convergence (K≤3 fixed; the float DDGI `ClampBounceCount` posture). Specular/glossy
indirect (diffuse SH only). A float bounce variant (strict integer — the GI2 headroom makes it sound). GI4 claims
ONLY: a deterministic integer multi-bounce feedback (K≤3), byte-identical HW==CPU, with the no-op + monotonicity
guards proving the bounce adds light without underflow or overflow, strict-zero cross-vendor — light that bounces,
deterministically.
