**Goal:** Create `engine/sim/hulljoint.h` (`hf::sim::hulljoint`): `JointedHullWorld` (a `gjk::HullWorld` + the
`joint::FxJoint` ball-joint list + the `joint::FxAngularLimit` hinge/cone list + the `hullfric::HullFrictionCache`)
and `StepJointedHullWorld(world, cfg)` = the HF3 friction step shell with the joint constraint passes inserted in the
PROVEN `joint::StepArticulatedContacts` order (integrate → K {ball | angular} joint passes → broadphase + the HF3
friction-contact warm solve → position de-pen → orient). Reuses `joint::SolveBallJoint`/`SolveAngularLimit` VERBATIM
over `world.bodies` (no new constraint math). Add `shaders/hulljoint_step.comp.hlsl` (the whole step single-thread,
int64 → Vulkan-only, chunked). Add `--hf4-joint-shot` (Vulkan) / `--hf4-joint` (Metal); prove the **chain stays
connected** + the **hinge holds its plane** + **GPU==CPU** bit-exact + determinism. Bake the **strict-zero** golden
`hf4_joint`.

## Design call: transplant the proven articulated step onto hull friction contacts; pin the order
`joint.h`'s `SolveBallJoint` (the anchor-coincidence impulse) and `SolveAngularLimit` (the swing-twist cone clamp) are
ALREADY body-index based over a `bodies` vector of `fpx::FxBody` — which is EXACTLY `gjk::FxBody` (a `using` alias),
the hull body type. And `joint::StepArticulatedContacts` ALREADY composes those joint passes with rigid contacts in
one tick. HF4 does NOT write new constraint math: it builds `JointedHullWorld` (the hull world + the joint lists) and
`StepJointedHullWorld` that runs the SAME `StepArticulatedContacts` structure but with the HF3 hull friction-contact
pass swapped for the fpx contact pass.
- **The composition ordering (the secondary crux — pin it):** integrate → **K {ball | angular} joint passes** (the
  position/velocity constraints) → broadphase + the HF3 friction+contact warm solve → position de-pen → orient.
  Joints first, then friction-contact velocity solve, then de-pen — the `joint::StepArticulatedContacts` order
  verbatim, so the joint pass and the friction pass never write the same body field in an order-ambiguous way within
  a sweep. FIXED joint-list order + FIXED `i<j` contact order + in-place mutation → two-run + GPU==CPU bit-identity
  by construction (the cloth/JT discipline).
- **The friction cache + the joints coexist:** the joints are stateless constraints (re-solved each tick); the
  friction cache persists (HF3). They compose in the one step with no shared mutable state beyond the bodies.

## Design decisions (locked)
1. **NEW `engine/sim/hulljoint.h`** (`hf::sim::hulljoint`, `#include`s hullfric/joint/warmhull/gjk/fpx READ-ONLY):
   - `struct JointedHullWorld { gjk::HullWorld hulls; std::vector<joint::FxJoint> joints; std::vector<joint::FxAngularLimit>
     limits; hullfric::HullFrictionCache cache; }`.
   - `struct JointedHullStepConfig { hullfric::HullFrictionStepConfig fric; int jointIters; }` (the HF3 friction
     config + the joint-pass iteration count).
   - `inline void StepJointedHullWorld(JointedHullWorld& w, const JointedHullStepConfig& cfg)` — the tick: integrate →
     `cfg.jointIters` sweeps of {`SolveBallJoint` over each `joint`, `SolveAngularLimit` over each `limit`} (FIXED
     list order) → the HF3 broadphase + friction+contact warm solve → de-pen → orient + cache update. Mirror
     `joint::StepArticulatedContacts` + HF3's friction pass.
   - `inline void StepJointedHullWorldN(w, cfg, ticks)` — chunked (1-tick/dispatch, TDR-safe).
   - `inline JointMeasure MeasureJointedHull(const JointedHullWorld& w)` — `MaxAnchorGap` (the largest ball-joint
     anchor separation — should be small after settling) + `SwingAngleCos` (the hinge swing vs its limit).
2. **NEW `shaders/hulljoint_step.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64, `:cs`): one thread runs the WHOLE
   `StepJointedHullWorldN` (`[numthreads(1,1,1)]`, the `hullfric_step.comp` convention; chunked 1-tick/dispatch).
   Writes the final bodies + cache. DXC `:cs` in `samples/hello_triangle/CMakeLists.txt` ONLY; the Vulkan-only
   comment in `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line.
3. **Showcase `--hf4-joint-shot <out>` (Vulkan) AND `--hf4-joint` (Metal) — WIRE BOTH (grep your OWN `visual_test.mm`
   for `--hf4-joint` BEFORE reporting DONE).** A PINNED scene composing BOTH new capabilities: a CHAIN of ~4 hulls
   linked by ball joints (hung from a static anchor — it should hang connected) + a hinged hull DOOR (an angular
   limit holding a swing axis) + a hull resting on the ground with friction. Stepped K ticks. Vulkan: dispatch
   `hulljoint_step.comp` → read back the final bodies; compute the CPU `StepJointedHullWorldN`; memcmp GPU==CPU
   strict; render a PURE-INTEGER side-view (the hanging chain + the door + the resting hull). Golden =
   `tests/golden/metal/hf4_joint.png` (CONTROLLER-baked).
4. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) GPU==CPU:** `hf4-joint: {bodies:<N>, joints:<J>, limits:<L>, ticks:<K>} GPU==CPU final-bodies BYTE-EQUAL`.
   - **(2) chain connected:** `hf4-joint: chain connected (maxAnchorGap:<g> small)` — after settling, the largest
     ball-joint anchor separation is below a small threshold (the chain did not come apart).
   - **(3) hinge holds:** `hf4-joint: hinge holds plane (swingCos>=limitCos)` — the hinged door's swing stays within
     its angular limit (the cone/hinge constraint held).
   - **(4) determinism + composition:** GPU/CPU two-run byte-identical; the resting hull's friction cone holds
     (`|jt|≤μ·jn`) WHILE the joints are satisfied (joints + friction + contacts composed in one deterministic step).
   - Golden discipline: ONLY `tests/golden/metal/hf4_joint.png`; do NOT commit it. Existing 255 goldens UNTOUCHED.
5. **Cross-backend bar:** strict **0.0000** (pure-integer side-view). Mac re-renders `--hf4-joint` vs the golden at
