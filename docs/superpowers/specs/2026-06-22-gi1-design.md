> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (RTX 4070, HW ray query). The FIRST slice of
> FLAGSHIP #29 (DETERMINISTIC LUMEN-CLASS GI, NEW header `engine/render/gi.h`, namespace `hf::render::gi`, slice-tag
> **GI**). Flagship #28 delivered deterministic hardware ray tracing (`rtrace.h`: integer `TraceClosest`/`TraceAnyHit`/
> `ShadeHitShadowed`, the HW candidate-drain `rt_query.comp`, the accel-struct RHI seam — all strict-zero

**Goal:** Create `engine/render/gi.h` (`hf::render::gi`): a `GiProbeGrid` (integer fx lattice), a host-precomputed
fx probe-ray direction table (`kGiRaysPerProbe = 16`, a Fibonacci sphere baked as Q16.16 constants — NO `cos`/`sin`
on the hot path), and `TraceProbeRays(grid, scene, out)` = per probe, per direction → `rtrace::TraceClosest` →
`rtrace::ShadeHitShadowed` (direct + RT shadow ray) → store the per-ray integer radiance (the BRUTE-FORCE no-cull
reference — the GI completeness oracle). Add `shaders/gi_probe_trace.comp.hlsl` (Vulkan inline RayQuery, the SAME
candidate-drain as `rt_query.comp`, one thread per probe). Add `--gi1-probe-shot` (Vulkan) / `--gi1-probe` (Metal)
over a small enclosure scene + a probe grid; prove the HW per-probe radiance buffer is **byte-identical** to the CPU
reference, and render a deterministic probe-radiance visualization. Bake the **strict-zero** golden `gi1_probe`.

## Design call: re-host the DDGI probe grid on the RT closest-hit, in integer
The merged DDGI traced probe rays with a float screen-space depth march (`probegi::TraceRayToDepth`) — view-
dependent, blind to off-screen geometry, and float. GI1 replaces that with `rtrace::TraceClosest` over the
procedural-AABB scene: world-space, off-screen-complete, and bit-exact (the HW BVH is ONLY a candidate generator;
OUR `(t,primIndex)` integer min owns the hit, so the result is independent of vendor traversal order — the RT2
contract). Each hit is shaded with the EXISTING integer `rtrace::ShadeHitShadowed` (direct Lambert + an RT shadow
ray via `TraceAnyHit`) — per-ray shadowing the float DDGI never had. The per-ray radiance is integer Q16.16 → the
whole probe-trace is strict-zero cross-vendor. GI1 produces ONLY the per-probe-per-ray radiance (GI2 encodes it to
integer SH; GI3 interpolates; GI4 adds multi-bounce; GI5 occlusion; GI6 the lit capstone).
- **The direction table is host-precomputed.** `kGiRaysPerProbe = 16` Fibonacci-sphere directions, computed once in
  `double` at table-definition time and baked as exact `fx` literals (a `static constexpr FxVec3 kGiProbeDirs[16]`)
  — so the GPU never evaluates a transcendental on the hot path and both backends read identical bits. (The
  `probe_sh.h` host-table discipline, in integer.)
- **The shade reuses `ShadeHitShadowed` verbatim** — direct light + RT shadow. A probe ray that MISSES → the
  scene background / a fixed sky term (a probe sees the sky). No bounce yet (GI4).

## Design decisions (locked)
1. **NEW `engine/render/gi.h`** (`hf::render::gi`), the flagship's append-only header (frozen per slice). Contents:
   - `struct GiProbeGrid { math::FxVec3 origin; fx spacing; int32_t nx, ny, nz; }` + `inline int ProbeCount(...)` +
     `inline FxVec3 ProbePos(grid, ix, iy, iz)` / `ProbePos(grid, linearIndex)` (integer lattice; probe `i` at
     `origin + (ix,iy,iz)*spacing`).
   - `inline constexpr int kGiRaysPerProbe = 16;` + `static constexpr math::FxVec3 kGiProbeDirs[16] = { ... }` (the
     baked Fibonacci-sphere fx unit directions — compute in `double`, write the Q16.16 literals; document the
     generator in a comment).
   - `struct GiRadiance { fx r, g, b; }` (Q16.16 linear radiance per ray) — the per-`(probe,ray)` output element.
     The buffer is `ProbeCount * kGiRaysPerProbe` `GiRadiance` (std430-compatible padding for the GPU mirror —
     pad to 16 B).
   - `inline void TraceProbeRays(const GiProbeGrid& grid, const rtrace::RtScene& scene, std::span<GiRadiance> out)`
     — per probe `p`, per dir `d`: `RtRay ray{ProbePos(grid,p), kGiProbeDirs[d]}`; `RtHit hit = TraceClosest(ray,
     scene)`; if hit: `occluded = TraceAnyHit({hit.pos + n*kRtShadowEps, scene.lightDir}, scene, kRtShadowMinT)`,
     `rgba = ShadeHitShadowed(hit, scene, occluded)`, unpack to `GiRadiance`; else the sky/background radiance.
     Brute-force, NO cull (the completeness oracle GI's HW path is checked against). Pure integer, deterministic.
   - `inline void GiProbesToImage(grid, out, std::span<uint32_t> img, w, h)` — a deterministic VIZ: lay the probes
     out as a 2D tile grid, each tile colored by the probe's MEAN per-ray radiance (integer average), for the
     golden. (A flat grid viz, not a 3D render — GI6 is the lit capstone.) Strict integer.
   - `inline rtrace::RtScene BuildGi1Scene(...)` (+ owning storage struct) — a small enclosure: a floor + a couple
     of colored wall AABBs + a sphere + a directional light (a Cornell-ish box so GI4's multibounce later shows
     color bleed). PINNED.
2. **NEW `shaders/gi_probe_trace.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq`): one thread per probe.
   For each of the 16 baked directions, open a `RayQuery` over `gTlas` (the candidate-drain + fx `TraceClosest`
   from `rt_query.comp`), shade with the inline `ShadeHitShadowed` + a shadow `RayQuery`, write the `GiRadiance` to
   `gRadiance[probe*16 + dir]`. The 16 directions are a shader constant array matching `kGiProbeDirs`. Register a
   DXC `csrq` line in `samples/hello_triangle/CMakeLists.txt` ONLY; the Vulkan-only comment block in
   `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line (Metal runs the CPU `TraceProbeRays`).
   - **TDR:** one thread per probe × 16 rays × (1 closest + 1 shadow) over ~8 prims — a small probe grid (e.g.
     `4×4×4=64` probes) is trivially under the TDR watchdog ([[hazard-forge-gpu-tdr-chunking]]); single dispatch, no
     chunking. (If a larger grid is ever used, one thread per `(probe,ray)` keeps it parallel.)
3. **Showcase `--gi1-probe-shot <out>` (Vulkan) AND `--gi1-probe` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--gi1-probe` BEFORE reporting DONE; [[hazard-forge-metal-showcase-gate]]).** Build
   `BuildGi1Scene()` + a pinned `GiProbeGrid` (4×4×4). Vulkan (HW): build the accel structure (reuse the
   `--rt2-query-shot` BLAS/TLAS build), dispatch `gi_probe_trace.comp`, read back the radiance buffer; compute the
   CPU `TraceProbeRays` reference; memcmp HW==CPU strict; render `GiProbesToImage` and write it. No-RT → CPU
   fallback + `hwRayQuery:false → SW`. Metal: CPU `TraceProbeRays` + `GiProbesToImage`. Golden =
   `tests/golden/metal/gi1_probe.png` (CONTROLLER-baked).
4. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) HW==CPU:** `gi1-probe: {probes:<P>, rays:<R>, lit:<L>} HW==CPU radiance BYTE-EQUAL (hwRayQuery:true)` —
     `std::memcmp` of the GPU radiance readback vs the CPU `TraceProbeRays` == 0.
   - **(2) probes see light:** `gi1-probe: probes lit (lit:<L> in (0, all), shadowed-probe darker)` — at least one
     probe-ray hit a lit surface AND at least one was shadowed (the per-ray shadow actually fired); not all-dark,
     not all-lit.
   - **(3) determinism:** CPU `TraceProbeRays` two-run byte-identical; HW two-run byte-identical.
   - **(4) no-op:** a `0`-probe grid → `ProbeCount==0` → zero dispatch → the radiance buffer untouched (the DDGI
     `ProbeDispatchGroups→0` discipline).
   - Golden discipline: ONLY `tests/golden/metal/gi1_probe.png`; do NOT commit it. Existing 246 goldens UNTOUCHED.
5. **Cross-backend bar:** strict **0.0000** (integer). Mac re-renders `--gi1-probe` vs the golden at 0.0000; the
   controller records the Vulkan-HW vs Metal cross-vendor diff (expect 0.0000).
6. **Tests — NEW `tests/gi_probe_test.cpp` (pure CPU, ASan-clean), ctest 119→120:** the 16 baked directions are
   unit-length within an fx epsilon and distinct; `TraceProbeRays` analytic (a probe facing a lit surface sees
   radiance > 0; a probe whose ray to the light is blocked is darker; a probe in open space → sky); two-run
   determinism; the `ProbeCount==0` no-op. Clean under `windows-msvc-asan`.
7. **Introspect.** Add EXACTLY `gi1-probe` (features) + `--gi1-probe-shot` (showcases) + update
   `tests/introspect_test.cpp`. CONTROLLER rebakes the JSON golden (verify it stages ` M `).

## Out of scope (YAGNI)
SH encode (GI2), interpolation (GI3), multi-bounce (GI4), occlusion weighting (GI5), the lit GI capstone (GI6).
Probe relocation / world-scale streaming / runtime re-bake (static bake per shot, like DDGI). Glossy/specular GI
(diffuse probes; specular stays on the RT4 mirror path). Denoising, temporal accumulation. A float Tier-B GI variant
(the whole arc is strict-integer — the moat). GI1 claims ONLY: a world-space, off-screen-complete, per-ray-shadowed
integer probe trace over the RT closest-hit, byte-identical HW==CPU on Vulkan HW, strict-zero cross-vendor — the
deterministic foundation the SH/multi-bounce GI is built on.
