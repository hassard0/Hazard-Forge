**Goal:** Append to `engine/anim/ik.h`: `struct IkSimState { IkBasePose base; IkRig rig; IkPose pose; }` (the base pose
+ the rig whose `target` is the moving state + the current solved pose — or just carry the target + recompute the pose
each tick), `struct IkCommand { int tick; int chain; FxVec3 delta; }` (kCmdMoveTarget — the per-tick target nudge),
`SimIkTick(state, parents, commands, tick)` (apply this tick's target moves, then `SolveRigToTargets` → the corrected
pose — re-derived from the moved target), `SnapshotIkRig`/`RestoreIkRig` (bit-exact round-trip of the mutable state —
the target + the solved pose), `IkStatesEqual`, `RunIkLockstep` (two peers from the same init + stream →
`IkStatesEqual` every tick; returns the converged authority state), and `RunIkRollback` (advance to `rollbackAt`,
snapshot, mispredict ≤3 ticks with a WRONG target stream that diverges the pose, restore, re-sim the correct stream;
corrected == authority + the mispredict diverged). Add `--ik5-lockstep-shot` (Vulkan, pure-CPU proof) / `--ik5-lockstep`
(Metal) rendering the converged authority pose. Bake the **strict-zero** golden `ik5_lockstep`.

## Design call: the AC5/VD5 lockstep mold over SolveRigToTargets; the target is the state, the pose is the pure function
The lockstep/rollback machinery is proven five times (FPX5, JT5, VH5, AC5, VD5). IK5 reuses the SHAPE verbatim, with
the IK-specific state being minimal and clean:
- **The replayable state = the effector target(s)** (`IkRig.target`, moved by `kCmdMoveTarget` each tick). The base
  pose + skeleton parents + chain topology are CONST (carried as params, re-supplied at restore, NOT snapshotted). The
  IK-corrected pose is a PURE FUNCTION of (base, target) — re-derived each tick by `SolveRigToTargets` (IK4, verbatim).
- **`SimIkTick` = apply target moves → `SolveRigToTargets`.** Deterministic by construction (fixed command order,
  fixed FABRIK iters, integer). Two peers running the same `SimIkTick` over the same command stream are byte-identical;
  rollback restores the exact target + re-sims.
- **The snapshot captures the target + the solved pose** (the `IkPose` localR). The pose is technically re-derivable
  from the target, so the snapshot is small; capturing the pose makes `IkStatesEqual` a direct byte-compare of the
  whole rig state (the VD5 whole-world-equality shape). `RestoreIkRig` restores both.
- **Determinism by construction:** `SolveRigToTargets` is the frozen IK4 integer solve (GPU==CPU already proven). The
  lockstep is a determinism PROPERTY of it — PURE CPU, no GPU dispatch, no shader, no TDR. The converged pose is
  bit-identical Vulkan-Windows AND Metal-Mac (both run the same CPU `SimIkTick`) → the cross-backend zero-diff golden
  IS the lockstep evidence.

## Out of scope (YAGNI)

warm-start / temporal coherence across ticks (the pose is re-derived from the target each tick — a future refinement
could seed from the previous pose, making the snapshot's pose load-bearing). Multiple competing rigs. The lit 3D render
(IK6). IK5 claims ONLY: the IK-driven rig state is bit-identical across two peers from the input target stream AND
rollback-replayable, pure CPU, strict-zero cross-vendor — deterministic replayable character animation.
