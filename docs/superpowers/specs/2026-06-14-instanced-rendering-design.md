# Slice Q — GPU Instanced Rendering (design)

Date: 2026-06-14
Branch: `slice-instancing`

## Goal
Draw a large field of meshes (>=144) that share one vertex/index buffer in a SINGLE draw call,
each placed by its own per-instance model matrix supplied through a second, per-instance vertex
stream. Keep the RHI seam clean and the existing pipelines/shaders/goldens untouched; add a
SEPARATE instanced pipeline + shader + golden.

## RHI additions (pure C++ interface, `engine/rhi/rhi.h`)
1. `GraphicsPipelineDesc.instanceLayout` — a `VertexLayout`. When its `attributes` is non-empty the
   pipeline declares a SECOND vertex binding (binding 1) whose input rate is per-instance. Binding 0
   stays the existing per-vertex `vertexLayout`. No new struct types are needed; the per-instance
   stride/attributes ride the existing `VertexLayout` type.
2. `ICommandBuffer::BindInstanceBuffer(IBuffer&)` — binds a buffer to vertex binding 1 (the
   per-instance stream). Default body is empty so backends/passes that never instance are unaffected.
3. `ICommandBuffer::DrawIndexedInstanced(indexCount, instanceCount, firstIndex=0, vertexOffset=0,
   firstInstance=0)` — instanced indexed draw. Default body forwards to `DrawIndexed` once
   (instanceCount ignored) so unrelated backends still link; both shipping backends override it.

The existing non-instanced `BindVertexBuffer` / `Draw` / `DrawIndexed` paths are untouched.

## Per-instance layout
A mat4 instance transform = 4 `RGBA32_Float` attributes at locations **7,8,9,10**, offsets
0/16/32/48, **stride 64**. (Locations 5/6 are reserved by the skinned vertex layout; 7-10 avoid any
collision.) The 16 floats are written column-major — byte-identical to the push-constant model
matrix the lit pipeline already uses — so `mul(model, pos)` in HLSL behaves the same whether the
matrix arrives via push constant or instance stream. No per-instance tint is added (kept minimal;
mesh vertex color + texture provide variation).

## Backends
### Vulkan (`engine/rhi_vulkan/`)
- `vulkan_pipeline.cpp`: when `instanceLayout.attributes` is non-empty, add a 2nd
  `VkVertexInputBindingDescription` (binding 1, `VK_VERTEX_INPUT_RATE_INSTANCE`) and append its
  attributes (each `VkVertexInputAttributeDescription.binding = 1`).
- `vulkan_command_buffer.cpp`: `BindInstanceBuffer` -> `vkCmdBindVertexBuffers(cmd, 1, 1, &vb, &0)`;
  `DrawIndexedInstanced` -> `vkCmdDrawIndexed(cmd, indexCount, instanceCount, firstIndex,
  vertexOffset, firstInstance)`.

### Metal (`engine/rhi_metal/`)
- New flat buffer slot `kVbInstance = 4` (past vertex 0 / frameUbo 1 / pushConst 2 / jointPalette 3)
  in `metal_common.h`. The generated instanced MSL reads the instance attributes from `stage_in`, so
  the instance buffer is a second vertex-descriptor buffer at index 4.
- `metal_pipeline.mm`: when `instanceLayout` is non-empty, add its attributes to the
  `MTLVertexDescriptor` with `bufferIndex = kVbInstance`, and set
  `layouts[kVbInstance].stepFunction = MTLVertexStepFunctionPerInstance`, stride = instance stride.
- `metal_command_buffer.mm`: `BindInstanceBuffer` -> `setVertexBuffer:offset:0 atIndex:kVbInstance`;
  `DrawIndexedInstanced` -> `drawIndexedPrimitives:...instanceCount:baseVertex:baseInstance:`.

## Shader (`shaders/lit_instanced.vert.hlsl`)
Copy of `lit.vert.hlsl` but the model matrix comes from 4 instance attributes (loc 7-10) assembled
into a `float4x4` instead of the push constant. The matrix columns map: attr7=col0 .. attr10=col3
(column-major, matching the CPU write). Reuses the UNCHANGED `lit.frag.hlsl` as the fragment stage.
No push constant for the model matrix; a small push constant still carries `float4 material`
(metallic/roughness) so the shared fragment's per-draw material path is fed (HF_MSL_GEN keeps the
same Frame/PushC binding scheme as lit.vert). MSL generated by the existing glslc->spirv-cross rule.

## Showcase + verification
- Vulkan `hello_triangle --instanced-shot <bmp>`: ground plane + sky + a 12x12 = 144 field of
  `Mesh::Sphere` instances drawn in ONE `DrawIndexedInstanced`. Each instance matrix is built
  deterministically from its index (grid x/z, a per-index height bob via sin, a per-index Y rotation)
  — NO randomness, golden-stable. Lit + shadowed (the field is also drawn instanced into the shadow
  map via a separate instanced depth-only shadow pipeline using `shadow_instanced.vert.hlsl`).
  Fixed camera/light. One frame -> 32-bit BMP. Converted to PNG and visually inspected.
- Metal `visual_test --instanced <png>`: same showcase; new golden `tests/golden/metal/instanced.png`
  if the Mac is reachable, else "Metal golden pending".

## Constraints honored
- Seam grep (`vk[A-Z]|MTL|Metal` over rhi/scene/math/asset/render) stays at the master baseline (12);
  zero new backend symbols above the backend dirs.
- Existing pipelines/shaders/goldens (`scene_shadow.png`, `skinning.png`, `pbr_helmet.png`) untouched.
- `ctest` stays green; a small unit test builds the instance buffer layout + a CPU instance matrix.
