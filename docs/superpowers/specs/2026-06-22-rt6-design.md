## Design call: the hero shot stays strict integer — the moat holds to the last pixel
Every prior RT slice is bit-exact HW==SW==CPU and strict-zero cross-vendor. A float capstone would introduce the ONE
non-deterministic slice and weaken the headline. Instead RT6 adds the one missing piece of visual polish — a **graded
sky** — as an INTEGER function of the ray direction, and renders a curated, higher-resolution hero scene with the full
feature set. The result is a beautiful ray-traced image that is STILL byte-identical across vendors. The marketing
line lands: "hardware ray tracing whose every frame — hero shot included — is bit-identical on NVIDIA and Apple."
- **The sky gradient** `SkyGradient(dir)`: normalize `dir` (fx, `RtNormalize`), map `t = (n.y + kOne)/2` to [0,kOne],
  lerp two integer colors (a warm horizon → a cool zenith) per channel in fx → RGBA8. Pure integer, deterministic.
  Replaces `scene.background` for misses (primary AND reflection). A directional fixed gradient — no time/animation.
- **Same one-bounce reflection + hard shadows as RT4/RT3** (reused verbatim) — RT6 changes ONLY the miss color (flat
  → graded sky), the scene, and the resolution. The reflective floor now mirrors a graded sky + the spheres.
- **480×360** (1.5× the 320×240 working size) for a crisper hero. One thread per pixel, ≤~3 rays/pixel over ~20
  prims → a single dispatch is FAR under the TDR watchdog ([[hazard-forge-gpu-tdr-chunking]]); no chunking.

## Design decisions (locked)
1. **APPEND to `rtrace.h`** (RT1+RT3+RT4 byte-frozen):
   - `inline uint32_t SkyGradient(const FxVec3& dir, ...)` — integer horizon→zenith lerp by `RtNormalize(dir).y`. Pin
     the two endpoint colors as constants (e.g. horizon `(120,110,140)`-ish warm-grey, zenith the RT2 background
     `(34,40,56)` deep blue) so the scene + shader agree. Returns RGBA8.
   - `inline uint32_t RenderSceneHero(scene, cam, w, h, out)` — IDENTICAL to `RenderSceneReflected` EXCEPT every
     place that returned `scene.background` on a miss now returns `SkyGradient(rayDir)` for the relevant ray (the
     primary miss uses the primary dir; the reflection miss uses the reflection dir). Returns a small struct or packs
     two counts (shadowed + reflective) for the proofs — or just returns the reflective count and the showcase also
     calls the RT3/RT4 helpers for the shadowed count. Strict integer.
   - **Keep `ShadeHitShadowed` unchanged** — it already returns `background` only via the caller; RenderSceneHero
     substitutes the sky at the call site. (Do NOT modify `ShadeHitShadowed`/`ShadeHitInt` — they read
     `scene.background`; instead, in RenderSceneHero, set `scene.background = SkyGradient(primaryDir)` is WRONG (it
     varies per ray). Compute the miss color explicitly: if `hit` is a miss → `out = SkyGradient(primaryDir)`; the
     shadow/reflection paths likewise compute their miss via SkyGradient. PIN this exactly so the shader matches.)
2. **NEW `shaders/rt_hero.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq`): the `rt_reflect.comp` body
   with the inline integer `SkyGradient(dir)` for misses. DXC `csrq` line in `samples/hello_triangle/CMakeLists.txt`
   ONLY; Vulkan-only comment in `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line.
3. **Showcase `--rt6-hero-shot <out>` (Vulkan) AND `--rt6-hero` (Metal) — WIRE BOTH (grep your OWN `visual_test.mm`
   for `--rt6-hero` BEFORE reporting DONE).** A curated hero scene at **480×360**: the reflective ground + a row/arc
   of colored spheres + the mirror sphere (`kRtMirrorSpherePrim`) + the two boxes, a nice 3/4 camera + the
   directional light, under the graded sky. Vulkan (HW): build the accel structure, dispatch `rt_hero.comp`, read
   back; CPU `RenderSceneHero`; memcmp HW==CPU strict; write the HW image. No-RT → CPU fallback. Metal: CPU
   `RenderSceneHero`. Golden = `tests/golden/metal/rt6_hero.png` (CONTROLLER-baked).
4. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) HW==CPU:** `rt6-hero: {rays:<R>, prims:<P>, shadowed:<S>, reflective:<Rc>} HW==CPU image BYTE-EQUAL (hwRayQuery:true)`.
   - **(2) full feature set present:** `rt6-hero: hero ok (shadowed>0, reflective>0, sky graded)` — both shadows AND
     reflections contributed, and the sky is non-flat (≥2 distinct background colors among the miss pixels).
   - **(3) determinism:** CPU two-run byte-identical; HW two-run byte-identical.
   - **(4) capability:** `hwRayQuery:true` (no-HW fallback prints `hwRayQuery:false → SW`).
   - Golden discipline: ONLY `tests/golden/metal/rt6_hero.png`; do NOT commit it. Existing 245 goldens UNTOUCHED.
5. **Cross-backend bar:** strict **0.0000** (integer). Mac re-renders `--rt6-hero` vs the golden at 0.0000; controller
   records the Vulkan-HW vs Metal cross-vendor diff (expect 0.0000).
6. **Tests — NEW `tests/rt_hero_test.cpp` (pure CPU, ASan-clean), ctest 118→119:** `SkyGradient` monotonic in `dir.y`
   (zenith ray vs horizon ray differ; two distinct directions → distinct colors); `RenderSceneHero` matches
   `RenderSceneReflected` on a scene with NO misses (sky never sampled) [or: equals it when background is forced to a
   flat color — pick the cleanest invariant]; the hero scene has shadowed>0 AND reflective>0; two-run determinism.
   Clean under `windows-msvc-asan`.
7. **Introspect.** Add EXACTLY `rt6-hero` (features) + `--rt6-hero-shot` (showcases) + update

## Out of scope (YAGNI)
Float shading / PBR / Fresnel (RT6 is strict integer — a float Tier-B fidelity pass is a GI-flagship #29 concern).
Multi-bounce reflections (ONE bounce, as RT4). Soft shadows, AO, GI (later / #29). Procedural sky beyond a vertical
gradient (no sun disc, no clouds). RT6 claims ONLY: a polished, higher-resolution hero render combining the full RT
feature set (primary + hard shadows + mirror reflections + graded sky), byte-identical HW==CPU on Vulkan HW, strict-
zero cross-vendor — the capstone that proves the whole HW-RT pipeline is deterministic. **Completes FLAGSHIP #28.**
