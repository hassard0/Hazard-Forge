# Slice AI4 — Deterministic AI: THE NPC AGENT IN THE GAMEPLAY TICK (sense → decide → query → path → act) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan + Mac+Metal. The FOURTH slice of the DETERMINISTIC
> AI flagship (GitHub issue #28, slice-tag **AI**). AI1–AI3 built the deterministic primitives (decision tree,
> environment queries, integer line-of-sight). AI4 COMPOSES them into a working NPC: an agent that **senses** (AI3 LOS
> to the player) → **decides** (AI1 tree: patrol vs chase) → **queries** (AI2: pick a destination) → **paths** (the
> frozen navmesh A*) → **acts** (emits a move), all in ONE deterministic pass in the fixed gameplay order. The agent is
> a `verdict` ECS entity; the whole update is bit-identical cross-platform. **APPENDS to `engine/ai/ai.h`** (`hf::ai`,
> AI1–AI3 byte-frozen — append-only). Header-only, pure CPU. Branch: `slice-ai4`. See [[hazard-forge-ai-roadmap]],
> [[hazard-forge-docs-style]], [[hazard-forge-build-env-vcvars]], [[hazard-forge-metal-showcase-gate]].

**Goal:** Append to `engine/ai/ai.h`: `struct AiAgent { int treeIndex; Blackboard bb; int navTarget; nav corridor
state; FxVec3 pos; ... }` (the per-agent AI state), and `StepAi(agents, navmesh, blockers, playerPos)` running ONE
deterministic agent update per agent in FIXED `order[]` sequence: (1) **perceive** — `LineOfSight(agent.pos,
playerPos, blockers)` + the integer distance to the player, written into `agent.bb`; (2) **decide** — `TickTree(tree,
agent.bb)` (the canonical tree: if player visible AND close → CHASE, else PATROL); (3) **query** — when the decision's
action says "pick a destination", `RunQuery(...)` (AI2) over the navmesh for the best cell (chase → toward the player's
last-seen cell; patrol → a ring point); (4) **path** — `nav::FindPath` from the agent's cell to the chosen
destination → a corridor; (5) **act** — set the agent's velocity / emit a move toward the next corridor centroid. A
`StepAiWorld` wrapper that runs `StepAi` then the FROZEN verdict systems (does NOT edit `verdict::StepWorld`). Add
`--ai4-agent-shot` (Vulkan) / `--ai4-agent` (Metal) rendering the agents + their decisions + their paths. Bake the
**strict-zero** golden `ai4_agent`.

## Design call: a parallel StepAiWorld (don't edit frozen StepWorld); fixed agent order; the agent IS a verdict entity
- **Compose, don't modify.** `verdict::StepWorld` (the gameplay tick) is byte-frozen. AI4 adds `StepAiWorld(world,
  navmesh, blockers, agents, playerPos)` that runs the AI pass (`StepAi`) THEN delegates to the frozen verdict systems
  (or just steps the agents' transforms) — a parallel composition, the IK4/IK5 "don't edit the frozen step" discipline.
  The agent's gameplay entity (its `verdict` Transform2D/BodyRef) is updated via the agent's chosen velocity/move; the
  AI state (`AiAgent`) is a parallel array keyed by the agent's `EntityId` (the `order[]`-keyed determinism contract).
- **The five-beat pass, fixed order.** Perceive → decide → query → path → act, in that fixed sequence, for each agent
  in `order[]` sequence. Each beat is the frozen AI1/AI2/AI3 primitive + the frozen nav A*. In-place integer mutation +
  fixed order → two-run + cross-backend bit-identity by construction.
- **The falsifiable decision beat:** the BT switches on PERCEPTION — when the player is visible (AI3 LOS true) AND
  within the chase distance, the tree's selector takes the CHASE branch (and the EQS picks a destination toward the
  player); when occluded or far, it falls back to PATROL. A test/proof toggles the player behind a blocker and shows
  the agent deterministically flips CHASE↔PATROL.
- **Within-band caveats (inherited):** nav A* within the largest connected component (AI2's scope); LOS is 2D
  AABB-only (AI3); the EQS "best" is a deterministic integer min. The composition adds no new approximation — it's the
  frozen primitives wired in a fixed order. Agent count is small (linear per-agent EQS + A*).
- **Docs-style:** generic terms ("agent", "perception", "decision", "destination query", "path") — no branding.
