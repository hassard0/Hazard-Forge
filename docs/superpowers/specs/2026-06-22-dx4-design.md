> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (backend-agnostic pure-CPU JSON/text goldens — NO
> Mac bake). The FOURTH slice of FLAGSHIP #31 (THE AGENT EXPERIENCE / AX PRODUCT, slice-tag **DX**). DX1–DX3 shipped
> the contract + read + authoring loop. DX4 productizes the **`hot-reload` capability** (enumerated in DX1, now
> realized) as a HEADLESS, DETERMINISTIC GOLDEN — today hot-reload runs only inside the interactive fly-editor
> (`FileWatcher` + a `LoadScene`-swap, main.cpp:10972-10995). DX4 lifts it into a `--hot-reload <A> <B>` driver and
> proves the load-time guarantee an agent needs: **a reloaded world is byte-for-byte identical to a cold load of the
> new scene** (no stale residue from the old scene), driven by the `FileWatcher`'s injectable-`StatFn` determinism
> hook (no real-filesystem mtime races). **APPENDS to `engine/runtime/hot_reload.{h,cpp}`** (`hf::runtime`, additive —
> `FileWatcher` byte-frozen). Branch: `slice-dx4`. See [[hazard-forge-ax-roadmap]], [[hazard-forge-introspect-golden-staleness]],
> [[hazard-forge-docs-style]], [[hazard-forge-build-env-vcvars]].

**Goal:** Factor the inline editor reload (main.cpp:10972-10995, `LoadScene` into a fresh registry) into a reusable
`hf::runtime::ApplyReload(ecs::Registry& reg, const scene::SceneResources& res, const std::string& path) ->
ReloadResult { bool loaded; int entityCount; bool equalToColdLoad; }` — it CLEARS `reg`, `LoadScene`s `path` into it,
and reports whether the result's `DumpScene` equals a cold `LoadScene(path)` into a fresh registry (a load is
state-equivalent to a cold load — no residue). Add a `--hot-reload <A.json> <B.json>` driver that: loads A (the
"before" world), uses a `FileWatcher` with an INJECTED stat to deterministically observe the scene path "change" to B,
`Poll()`s the change, `ApplyReload`s B, and emits a deterministic `reload_trace.json`
`{ watched, changed:[B], beforeCount, afterCount, equalToColdLoad, noResidue, dumpAfter }`. Register `--hot-reload` in
`kShowcases` (churns `contentHash`, `schemaVersion` HOLDS at 1 — the DX1 design, validated a fourth time). Bake the
**strict-zero** Windows byte-golden `tests/golden/agent/reload_trace.json`.

## Design call: factor the existing swap; prove reload == cold-load; use the injected StatFn for determinism (NO real mtime)
The live editor already does the reload (main.cpp:10972-10995): a `FileWatcher` watches the scene path, and on a
`Poll()` change it `LoadScene`s a fresh registry. DX4 makes that headless + deterministic + golden:
- **`ApplyReload` = clear-reg + `LoadScene` + equal-to-cold-load check.** Reloading is exactly loading into a cleared
  registry; the value is the GUARANTEE that this equals a cold load (no stale entities/components survive). The check:
  `DumpScene(reg after reload) == DumpScene(fresh registry cold-loaded from B)` — byte-identical.
- **The `FileWatcher` drives change-detection deterministically via the injected `StatFn`** (hot_reload.h:24 — the
  whole reason it exists): the driver supplies a fake stat returning a fixed pre-change mtime then a fixed post-change
  mtime, so `Poll()` reports the change WITHOUT any real-filesystem `last_write_time` race. The trace is therefore
  bit-reproducible run-to-run and backend-identical (verify once on Windows).
- **No-residue is the falsifiable beat:** load A (say 3 entities incl. a mesh only A uses), reload B (say 2 entities,
  none of A's), and assert A's entity set is GONE — `afterCount == B's count` and the dump has none of A's
  A-only contents. A reload that merely ADDED B's entities onto A's would fail this.
- **`equalToColdLoad` is the load-equivalence guarantee** an agent codes against: "reload(B) is indistinguishable from
  a fresh process that cold-loaded B."

## Design decisions (locked)
1. **APPEND to `engine/runtime/hot_reload.{h,cpp}`** (`hf::runtime`): `struct ReloadResult { bool loaded; int
   entityCount; bool equalToColdLoad; }` + `ReloadResult ApplyReload(ecs::Registry& reg, const scene::SceneResources&
   res, const std::string& path)`. hot_reload.cpp gains `#include "scene/scene_io.h"` + `"ecs/ecs.h"` (both hf_core,
   backend-agnostic — the header's "NO rhi/SDL/backend symbols" contract is preserved; these are intra-hf_core deps,
   and the `FileWatcher` itself is UNCHANGED). If the dependency direction is awkward (runtime→scene), the implementer
   MAY instead place `ApplyReload` in a NEW `engine/runtime/scene_reload.{h,cpp}` — document the choice; the deliverable
   (the function + the headless driver + the golden) is unchanged.
2. **`--hot-reload <A.json> <B.json>`** (Windows `samples/hello_triangle/main.cpp`): pure-CPU (sentinel resources, no
   GPU — the DX3 pattern). (a) `LoadScene(reg, res, A)` → `beforeCount` + capture A's dump; (b) a `FileWatcher` with an
   injected `StatFn` returning a fixed mtime for the watched path, `Watch(path)` at the pre-change value; (c) bump the
   injected mtime, `Poll()` → assert it reports `[path]` changed; (d) `ApplyReload(reg, res, B)` → `afterCount` +
   `equalToColdLoad`; (e) compute `noResidue` (A's entity count/contents absent); (f) write the deterministic
   `reload_trace.json` (LF-clean, no BOM). Wire the Metal `--hot-reload` in `metal_headless/visual_test.mm` for
   symmetry. **Grep your OWN visual_test.mm for `--hot-reload` before reporting DONE.** (If the main if/else hits MSVC
   C1061, use the DX1-3 pre-scan/fold workaround.)
3. **Register `--hot-reload` in `kShowcases`** (introspect.cpp). Churns `agentApi.contentHash`; `schemaVersion` STAYS
   1. Note before/after `contentHash`. (The `hot-reload` capability — enumerated since DX1 — is now realized.)
4. **The committed scenes `assets/scenes/dx4_a.json` + `dx4_b.json`** — two small DISTINCT valid scenes (different
   entity counts, A using a mesh B doesn't) referencing only registered resource names, so the no-residue + equal-to-
   cold-load proofs are non-trivial.
5. **PROOFS (exact stdout lines, fail loudly):**
   - `dx4-reload: watch(A) -> poll sees [B] changed -> reload -> {before:<N>, after:<M>}`
   - `dx4-reload equiv: post-reload DumpScene == cold LoadScene(B) BYTE-IDENTICAL`
   - `dx4-reload no-residue: A's entities absent after reload (after==B's count, A-only mesh gone)`
   - `dx4-reload determinism: two reload traces BYTE-IDENTICAL` + `reload_trace.json byte-golden MATCH`
   - Golden discipline: the implementer MAY commit `reload_trace.json` (deterministic Windows artifact). Do NOT rebake
     `agent_api.json` / `default_scene.json` — the CONTROLLER does. NO Metal golden.
6. **Tests — NEW `tests/hot_reload_test.cpp` (pure CPU, ASan-clean), ctest +1:** (the FileWatcher has no test yet —
   add one) inject a fake `StatFn` backed by an in-memory map; `Watch` baselines (first `Poll` empty); a bumped mtime
   → `Poll` reports the path; a missing→present transition counts as changed; a disappearance does not. Plus
   `ApplyReload`: build a `SceneResources` with sentinel named pointers, load A then `ApplyReload(B)` → `entityCount`
   == B's, `equalToColdLoad == true`, and the registry has none of A's A-only entities (no residue); two `ApplyReload`s
   byte-identical. Register in `tests/CMakeLists.txt`.

## Out of scope (YAGNI)
Shader/.spv hot-reload (hot_reload.h:6 mentions it — GPU, best-effort, not golden-able here). Live material reload
(already `--material-live-shot`). Real-filesystem mtime polling in the golden path (the injected `StatFn` is the
deterministic hook — a real-mtime test would be racy/non-reproducible). Incremental/diff reload (it's a full
clear+reload — the editor semantics). DX4 claims ONLY: a deterministic headless reload that is byte-equivalent to a
cold load (no residue), driven by the injectable watcher, byte-golden.
