## Design call: integer trilinear blend with partition-of-unity weights, narrow once
A query point in the probe lattice falls in a cell with a fractional offset `(fx, fy, fz)` per axis (Q16.16 in
`[0,kOne]`). The 8 corner weights are products of `(frac or kOne-frac)` per axis ‚Äî **integer**, and they sum EXACTLY
to `kOne` (partition of unity, a hard invariant the test pins). The blended SH is `Œ£_c weight_c * cornerSH_c` per
coeff per channel. Since `weight (‚â§kOne) * coeff (‚â§~2*kOne)` is a Q32.32 product that overflows int32, **accumulate
the 8-corner blend in an int64** and quantize to Q16.16 ONCE (the GI2 discipline). Then `FxSHEvaluate` gives the
irradiance. All integer ‚Üí bit-exact GPU==CPU, strict-zero cross-vendor.
- **Lattice-point identity:** a query exactly AT probe `p`'s position has `frac==0` on every axis ‚Üí weight 1.0 on
  corner `p`, 0 elsewhere ‚Üí the blend is EXACTLY probe `p`'s SH (a falsifiable proof the index/weight math is right).
- **Out-of-grid queries clamp** to the boundary cell (deterministic edge handling).
- **Shader path:** prefer MSL-native (`:cs` both lists) for GPU-on-both-backends; if the int64 `weight*coeff` product
  forces Vulkan-only (glslc can't parse int64), that is ACCEPTABLE ‚Äî the GI2 convention (Vulkan-only `:cs`, Metal
  runs the CPU reference, byte-identical by construction, cross-vendor strict-zero). The implementer picks based on
  what compiles + documents which. (Restructuring to an int32 accumulator via careful scaling is welcome but not
  required.)

## Design decisions (locked)
1. **APPEND to `gi.h`** (GI1+GI2 byte-frozen):
   - `struct FxProbeWeights { int32_t idx[8]; fx w[8]; }` (the 8 corner probe linear-indices + integer trilinear
     weights, `Œ£ w == kOne`). `inline FxProbeWeights FxNearestProbes(const GiProbeGrid& grid, const FxVec3& point)`
     ‚Äî floor-cell + frac + the 8 weights; clamp out-of-grid to the boundary.
   - `inline FxProbeSH FxInterpolateSH(const FxProbeWeights& w, std::span<const FxProbeSH> shBuffer)` ‚Äî int64
     8-corner blend, quantize once.
   - `inline GiRadiance FxInterpolateIrradiance(const GiProbeGrid& grid, std::span<const FxProbeSH> shBuffer,
     const FxVec3& point, const FxVec3& normal)` ‚Äî `FxSHEvaluate(FxInterpolateSH(FxNearestProbes(grid,point),
     shBuffer), normal)`.
   - `inline void InterpolateField(grid, shBuffer, std::span<const FxVec3> points, std::span<const FxVec3> normals,
     std::span<GiRadiance> out)` ‚Äî per query point.
   - `inline void GiFieldToImage(...)` ‚Äî the VIZ: a 2D slice through the probe volume (a horizontal plane at a fixed
     y), each pixel = `FxInterpolateIrradiance` at that world point for a fixed normal (+Y) ‚Üí a SMOOTH irradiance
     field (much smoother than the discrete GI1/GI2 probe grids). PINNED slice + resolution. Strict integer.
2. **NEW `shaders/gi_interp.comp.hlsl`** (one thread per query point; pure integer, NO RayQuery): reads the GI2 SH
   buffer + the query points/normals, runs `FxInterpolateIrradiance` VERBATIM, writes `GiRadiance` to `gOut[i]`.
   Register `:cs` in `samples/hello_triangle/CMakeLists.txt`; in `metal_headless/CMakeLists.txt` add the
   `hf_gen_msl` line IF the shader is int32-only (GPU both backends), ELSE the Vulkan-only comment block (int64 ‚Üí
   Metal CPU ref). Document which.
3. **Showcase `--gi3-interp-shot <out>` (Vulkan) AND `--gi3-interp` (Metal) ‚Äî WIRE BOTH (grep your OWN
   `visual_test.mm` for `--gi3-interp` BEFORE reporting DONE).** Build the GI1 scene + grid, `TraceProbeRays` +
   `EncodeAllProbes` (CPU), then build a dense 2D slice of query points; dispatch `gi_interp.comp`, read back;
   compute the CPU `InterpolateField` reference; memcmp GPU==CPU strict; render `GiFieldToImage`. Golden =
   `tests/golden/metal/gi3_interp.png` (CONTROLLER-baked).

## Out of scope (YAGNI)
Multi-bounce (GI4), occlusion weighting (GI5 ‚Äî the trilinear blend here is unoccluded; leak-fix is GI5), the lit GI

deterministic integer trilinear SH interpolation with partition-of-unity weights and lattice-point identity,
bit-exact GPU==CPU, strict-zero cross-vendor ‚Äî the continuous irradiance field the GI composite samples.
