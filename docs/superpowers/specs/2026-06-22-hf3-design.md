## Design call: the warmhull step with friction, cache persisted across ticks
HF2 solved a fixed manifold set in isolation. HF3 is the real tick: it re-implements the `StepWarmHullWorld` 5-pass
shell (warmhull is frozen — HF3 does NOT call it; it reproduces the structure with the friction solver):
1. **Predict-integrate** every dynamic body (`fpx::IntegrateBodyFull` + per-tick damping) — the WH3 pass, verbatim.
2. **Broadphase** the all-pairs `i<j` hull contacts (the frozen GJK/EPA narrowphase via the HF1
   `BuildAllHullFrictionManifoldsPairs`), building the tangent basis per pair this tick.
3. **Warm-seed** each manifold from the persistent `cache` (HF1 `MatchHullFrictionCache` — keyed warm-start, tangent
   cold-start on a basis-axis flip), then `cfg.solveIters` Gauss-Seidel sweeps of `SolveHullFrictionWarm` (HF2).
4. **Position de-penetration** (the WH3 linear push-apart along the A→B normal, verbatim).
5. **Orientation** integrate + write the accumulated impulses back into the `cache` (`UpdateHullFrictionCache`).
The cache PERSISTS across ticks — so a resting contact's friction impulse warm-starts each tick, the regime where
warm-starting matters (the HF2 lever compounds). All integer → GPU==CPU bit-exact.
- **The control is BEHAVIORAL, not cross-solver byte-identity.** (The HF2 friction solver's lineage differs from
  warmhull's normal-only solver, so `μ=0` is NOT guaranteed byte-identical to the frozen WH3 step.) Instead: `μ=0`
  is a frictionless HF3 run where the hull SLIDES off the ramp; `μ>0` is the SAME scene where it GRIPS and rests —
  the falsifiable friction delta (friction is the thing holding it). A `μ=0` HF3 run is two-run byte-identical
  (determinism), and the de-pen/integrate passes are the WH3 passes verbatim.
