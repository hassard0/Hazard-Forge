## Design call: the bit-exact IK runs on a small leg sub-chain (fits the IK structs); the Fox render is the existing float path
The JT6/AC6 capstones already drive the Fox `lit_skinned` render from a deterministic pose (`RagdollFromSkeleton` over
the Fox skeleton → `PoseToPalette` → `SetJointPalette` → the lit_skinned draw). IK6 swaps in an IK-corrected palette:
- **The bit-exact IK operates on a LEG SUB-CHAIN, not the whole Fox.** `IkBasePose` holds `kIkMaxJoints` (16) joints;
  the Fox skeleton may exceed that, and IK6 does NOT edit that frozen cap. Instead: pick a leg chain of ~3–4 Fox joints
  (root..foot — find them by joint name / a hardcoded Fox index set after inspecting `fox.skeleton`), capture that
  sub-chain's base LOCAL poses + the chain root's WORLD transform (from the Fox base FK) snapped to Q16.16 (the
  documented float crossing), run `SolveRigToTargets` (the frozen IK4 bit-exact solve) toward a world foot-target, and
  write the corrected leg-joint LOCAL rotations back into the Fox's FULL local pose (float). The leg chain ≤ 16 joints
  → fits the IK structs cleanly.
- **The render is the EXISTING float path** (the JT6/AC6/animation `lit_skinned` pipeline, REUSED VERBATIM — no new
  shader/RHI): the Fox's corrected full local pose → the float skinning palette (`PaletteFromLocalPose` / the engine's
  palette builder) → `SetJointPalette` → the lit_skinned + shadow_skinned draw from a fixed 3/4 camera + directional
  light. MATTE/standard Fox material (the existing Fox render look).
- **The bit-exact part = the leg IK solve** (integer, the IK1–IK5 within-band residual carries); the Fox base pose
  sample + the palette read-back + the final raster are float (render-only). So the golden is the FLOAT bar: per-backend
  Metal-baked golden, Metal==Metal-golden DIFF 0.0000 (deterministic two-run) + provenance + a DOCUMENTED in-band
  cross-vendor mean (~20–60, the float baseline, the MC5/FPX6/JT6/AC6 convention).
- **Compose-the-moat (a proof beat, keep it light):** show the IK-corrected pose can SEED the deterministic ragdoll —
  e.g. drive `active::ActiveRagdoll`/`joint::Ragdoll` qTargets from the IK-corrected pose (the `active.h` WriteClipTargets
  bridge) and confirm it settles, i.e. anim(IK) → ragdoll → recover all compose. If wiring the full ragdoll-recover
  into the render is heavy, prove the composition at the DATA level (the IK palette feeds the ragdoll bind) in the
  test, and keep the RENDER the IK-posed Fox money-shot. Do NOT over-scope — the render capstone is the deliverable.
