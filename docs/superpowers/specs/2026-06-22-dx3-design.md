# Slice DX3 — The Agent Experience Product: THE DECLARATIVE SCENE-SPEC AUTHORING LOOP — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (backend-agnostic pure-CPU JSON goldens — NO Mac
> bake). The THIRD slice of FLAGSHIP #31 (THE AGENT EXPERIENCE / AX PRODUCT, slice-tag **DX**). DX1 shipped the
> versioned contract; DX2 the scene-query read. DX3 productizes the **`author` capability** (enumerated in DX1, now
> realized): a closed authoring loop where an agent writes a scene as a declarative JSON **spec**, the engine builds it
> deterministically (`LoadScene`), re-emits it canonically (`DumpScene`), and the **round-trip is byte-identical** —
> proving an agent can author a scene as text and the engine reproduces it bit-for-bit, with a stable canonical form
> (the spec is a fixed point under canonicalization). The float engine can't offer this as a guarantee: a non-canonical
> or non-deterministic loader breaks the "this spec → this exact scene" contract. **APPENDS to
> `engine/scene/scene_io.{h,cpp}`** (`hf::scene`, additive — `LoadScene`/`DumpScene` byte-frozen). Branch: `slice-dx3`.
> See [[hazard-forge-ax-roadmap]], [[hazard-forge-introspect-golden-staleness]], [[hazard-forge-docs-style]],
> [[hazard-forge-build-env-vcvars]].

**Goal:** Add `editor`/`scene`-level `CanonicalizeScene(specPath, reg, resources) -> std::string` = `LoadScene(spec)`
then `DumpScene` (the existing pretty-printer, `%g` floats, 2-space indent — scene_io.cpp) returning the canonical
scene JSON; plus a file-free `CanonicalizeSceneText(specJson, reg, resources)` overload for the unit test. Add a
`--author-scene <spec.json> <out.json>` driver that writes the canonical result. Commit a spec `assets/scenes/dx3_spec.json`
(intentionally NON-canonical input — keys in a different order / extra whitespace) and the canonical golden
`tests/golden/agent/authored_scene.json`. Register `--author-scene` in `kShowcases` (so it's discoverable; this churns
`agentApi.contentHash`, `schemaVersion` HOLDS at 1 — the DX1 design, validated a third time). → the deterministic
text→scene→golden authoring loop.

## Design call: round-trip canonicalization = LoadScene∘DumpScene; the spec is a fixed point; reuse the frozen IO verbatim
`scene_io` already has the two halves: `LoadScene` (name→pointer resolve, scene_io.h:77) and `DumpScene` (pointer→name
reverse + the JSON pretty-printer, scene_io.h:83), and it is deliberately render-symbol-free (opaque `SceneResources`
pointers). DX3 composes them into a canonicalizer and proves the fixed-point property:
- **`CanonicalizeScene` = `DumpScene(LoadScene(spec))`** — a pure function. The INPUT spec may be non-canonical
  (arbitrary key order, whitespace, optional fields omitted); the OUTPUT is the engine's single canonical form.
- **The fixed-point (idempotence) proof:** `Canonicalize(Canonicalize(spec)) == Canonicalize(spec)` — feeding the
  canonical output back through the loop yields byte-identical bytes. This is the property that makes "this spec → this
  scene" a *contract*: there is exactly one canonical text for a given scene, and the loop converges to it in one step.
- **Determinism:** `DumpScene` already emits a FIXED key order + `%g` floats (scene_io.cpp), so two canonicalizations
  of the same spec are byte-identical and backend-identical (verify once on Windows — the `showcase3_graph.json`
  material-golden precedent, verify.ps1:477–484).
- **Error path:** an unknown mesh/texture name (or malformed spec) → `LoadScene`'s existing deterministic error → the
  driver prints a deterministic error line and exits NONZERO (the spec is rejected, not silently half-built).

## Design decisions (locked)
1. **APPEND to `engine/scene/scene_io.{h,cpp}`** (`hf::scene`): `std::string CanonicalizeScene(const char* specPath,
   ecs::Registry& reg, const SceneResources& res)` (load the spec file → dump) + `std::string
   CanonicalizeSceneText(const char* specJson, ecs::Registry& reg, const SceneResources& res)` (the in-memory overload
   for the test). Both pure; both throw `std::runtime_error` on a malformed spec (the `LoadScene` contract). NO render
   symbols.
2. **`--author-scene <spec.json> <out.json>`** (Windows `samples/hello_triangle/main.cpp`): set up the scene resources
   (the same named meshes/textures the default scene registers), `LoadScene(spec)` into a fresh `ecs::Registry`,
   `DumpScene`, write to `<out.json>` LF-clean (no BOM). On a `LoadScene` throw / unknown name: print a deterministic
   `author-scene: error: <msg>` line and exit nonzero. Wire the Metal `--author-scene` in `metal_headless/visual_test.mm`
   for symmetry (bytes backend-identical). **Grep your OWN visual_test.mm for `--author-scene` before reporting DONE.**
3. **Register `--author-scene` in `kShowcases`** (introspect.cpp) with a one-line description. This churns
   `agentApi.contentHash`; `schemaVersion` MUST stay 1. Note the before/after `contentHash` in your report. (The
   `author` capability — enumerated since DX1 — is now realized; no new `agentApi.commands` entry, it's a CLI authoring
   driver, not a scene-command-bus op.)
4. **The committed spec `assets/scenes/dx3_spec.json`** — a small valid scene authored as text, intentionally
   NON-canonical (e.g. object keys in a non-`DumpScene` order, an optional field omitted to default), referencing only
   meshes/textures the default resources register, so the round-trip + idempotence proofs are non-trivial (the input
   bytes ≠ the canonical golden bytes, but `Canonicalize(input)` == `Canonicalize(golden)` == the golden).
5. **PROOFS (exact stdout lines, fail loudly):**
   - `dx3-author: spec authored -> {entities:<N>} canonical byte-golden MATCH`
   - `dx3-author round-trip: LoadScene(spec) -> DumpScene == golden BYTE-IDENTICAL`
   - `dx3-author idempotent: Canonicalize(Canonicalize(spec)) == Canonicalize(spec) (fixed point)`
   - `dx3-author unknown-mesh: spec naming 'nope' -> deterministic error, exit nonzero`
   - Golden discipline: the implementer MAY commit `authored_scene.json` (deterministic Windows artifact). Do NOT
     rebake `agent_api.json` / `default_scene.json` — the CONTROLLER does (the new showcase + new `contentHash`). NO
     Metal golden.
6. **Tests — NEW `tests/agent_author_test.cpp` (pure CPU, ASan-clean), ctest +1:** build a `SceneResources` with a few
   fake-but-named resource pointers (the opaque-pointer contract lets the test register non-null sentinels under names
   like `"cube"`/`"duck"`); `CanonicalizeSceneText` of a small spec == the expected canonical text; idempotence
   (canonicalize twice == once); round-trip determinism (two calls byte-identical); an unknown mesh name throws
   `std::runtime_error`. Register in `tests/CMakeLists.txt`.

## Out of scope (YAGNI)
Authoring non-render entities (gameplay/sim — the verdict world, a later concern). Materials graphs (already
`--material-introspect`). A spec schema validator / linter beyond `LoadScene`'s existing name/field checks. Partial /
diff authoring (the bus `add`/`set_*` mutate ops already do incremental authoring). DX3 claims ONLY: a deterministic
declarative scene-spec → canonical-scene round-trip with a fixed-point guarantee and a deterministic reject path,
byte-golden.
