# Slice RT5 — Hardware Ray Tracing: DETERMINISM-ENVELOPE + LOCKSTEP TIE-IN (sim ⊗ RT) — Design

> Autonomous-session spec. Verifiable headlessly on Windows+Vulkan (RTX 4070, HW ray query). The FIFTH slice of
> FLAGSHIP #28 (HARDWARE RAY TRACING, `hf::render::rtrace`, slice-tag **RT**). RT1–RT4 built the deterministic SW
> tracer, the Vulkan + Metal HW ray query, hard shadows, and mirror reflections — all strict-zero cross-vendor.
> RT5 is the INTEGRATION/ENVELOPE slice: it COMPOSES the determinism moat (a `fpx` deterministic rigid-body sim,
> lockstep-replayable) with the RT render (the RT4 `rt_reflect.comp` HW path + the CPU reference), proving the
> combined system is **fully reproducible**: two lockstep peers fed only inputs produce byte-identical sim states,
> and the **HW-ray-traced image of the authority == the HW-ray-traced image of the replica == the CPU reference**.
> This closes the moat↔RT loop: *a deterministic physics simulation, hardware-ray-traced, is bit-identical between
> peers and across vendors.* **NO new shader (reuses `rt_reflect.comp`), NO `rtrace.h` change** — pure showcase
> composition + a sim→RT-scene converter in the showcase TU. `rtrace.h` BYTE-FROZEN. Branch: `slice-rt5`. See
> [[hazard-forge-raytracing-roadmap]], [[hazard-forge-fpx-roadmap]], [[hazard-forge-docs-style]],
> [[hazard-forge-introspect-golden-staleness]], [[hazard-forge-metal-showcase-gate]].

**Goal:** Add the showcase `--rt5-simrender-shot` (Vulkan) / `--rt5-simrender` (Metal): build a deterministic `fpx`
sim (a handful of spheres dropping + settling on a ground), run `fpx::RunLockstep` for the authority AND the replica
(same init + stream → bit-identical), convert the settled bodies → an `rtrace::RtScene` (each `FxBody` → an
`RtSphere`; a reflective ground AABB), RT-render via `rtrace::RenderSceneReflected` (CPU) + the `rt_reflect.comp` HW
path, and prove the closure (sim lockstep + RT(authority)==RT(replica) + HW==CPU + render is a pure read of the sim).
Bake the **strict-zero** golden `rt5_simrender` — the hardware-ray-traced settled physics scene.

## Design call: compose, don't extend — the sim and the RT render are both already deterministic
The whole point of RT5 is that NOTHING new is needed in the bit-exact core: `fpx` (the deterministic Q16.16 rigid-body
sim) and `rtrace::RenderSceneReflected` (the strict-integer RT render) are EACH already bit-exact and lockstep/
cross-vendor proven. RT5 just COMPOSES them and asserts the joint invariant. The sim→RT bridge is a trivial pure
function (`FxBody.pos`/`.radius` → `RtSphere.center`/`.radius`; `rtrace::FxVec3` IS `fpx::FxVec3`, same Q16.16 type),
defined in the showcase TU so `rtrace.h` keeps NO sim dependency. The HW render reuses `rt_reflect.comp` verbatim,
fed the sim-derived primitives.
- **The reflective floor mirrors the settled physics pile** — the ground AABB is `primIndex 0` (`ReflectivityFor=0.55`),
  the sim bodies are `primIndex 1..B` → the RT4 reflection + RT3 shadow path renders the settled spheres reflected
  in the floor, with shadows. Visually: a ray-traced physics result.
- **The envelope is eps_rt = 0** (strict): because RT1–RT4 are strict integer, `HW == SW == CPU` exactly, so the
  "determinism envelope" `visresolve(HW,SW) < eps_rt` collapses to byte-equality. RT5 states this concretely on a
  sim-driven scene (not a hand-built one), the strongest form.

## Design decisions (locked)
1. **NO `rtrace.h` change, NO new shader.** RT5 is showcase-level. The sim→RT converter
   `BodiesToRtScene(const fpx::FxWorld&, ...)` lives in the showcase TU(s) (main.cpp + visual_test.mm — a small pure
   function each, or a shared tiny header under `samples`/`metal_headless` if both need it; do NOT put it in
   `rtrace.h` or any engine header — keep the render core sim-free). It maps: ground AABB → `primIndex 0`; each
   `FxBody bi` → `RtSphere{center=bi.pos, radius=bi.radius, primIndex=1+i}`. Fixed camera/light (the RT2 camera).
2. **The deterministic sim.** A FIXED `fpx::FxWorld` (≈6–10 spheres dropped above a ground, varied x/z, `radius`
   set, the `--fpx-shot` gravity/dt/iters config) + a FIXED (possibly empty) `std::vector<fpx::FxCommand>` stream.
   `authority = fpx::RunLockstep(init, stream, N, dt, iters)`; `replica = fpx::RunLockstep(init, stream, N, dt, iters)`
   (same inputs, no shared state). N chosen so the pile SETTLES (a resting pile is the money visual).
3. **Showcase `--rt5-simrender-shot <out>` (Vulkan) AND `--rt5-simrender` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--rt5-simrender` BEFORE reporting DONE).** Both: run the sim lockstep ×2, convert the
   authority world → `RtScene`, render. Vulkan (HW): build the accel structure from the converted scene, dispatch
   `rt_reflect.comp` (HW), read back; CPU `RenderSceneReflected(authScene)`; memcmp HW==CPU strict; ALSO convert the
   replica world → `RtScene` and CPU-render it, memcmp RT(authority)==RT(replica); write the HW image. No-RT → CPU
   fallback. Metal: CPU `RenderSceneReflected`. Golden = `tests/golden/metal/rt5_simrender.png` (CONTROLLER-baked).
4. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) sim lockstep:** `rt5-simrender: {bodies:<B>, ticks:<N>} sim lockstep authority==replica BIT-IDENTICAL` —
     memcmp the two `fpx::FxWorld` body vectors (field-wise or the existing fpx equality).
   - **(2) RT render closure:** `rt5-simrender: RT(authority)==RT(replica) image BYTE-EQUAL` — the CPU RT image of
     the authority world == the CPU RT image of the replica world (a deterministic sim renders reproducibly).
   - **(3) HW envelope:** `rt5-simrender: HW==CPU image BYTE-EQUAL (eps_rt:0, hwRayQuery:true)` — the `rt_reflect.comp`
     HW image == the CPU `RenderSceneReflected` (the determinism envelope, eps_rt=0). No-HW path prints
     `hwRayQuery:false → SW`.
   - **(4) render is render-only:** `rt5-simrender: render-only (sim world byte-unmutated, statesEqual:true)` — the
     `fpx::FxWorld` is byte-identical before vs after the RT render (a pure read; RT does not mutate the sim).
   - Golden discipline: ONLY `tests/golden/metal/rt5_simrender.png`; do NOT commit it. Existing 244 goldens UNTOUCHED
     (RT is render-only — the fpx/sim goldens are byte-unchanged by construction).
5. **Cross-backend bar:** strict **0.0000** (integer). Mac re-renders `--rt5-simrender` vs the golden at 0.0000;
   controller records the Vulkan-HW vs Metal cross-vendor diff (expect 0.0000).
6. **Tests — NEW `tests/rt_simrender_test.cpp` (pure CPU, ASan-clean), ctest 117→118:** `fpx::RunLockstep`
   authority==replica on the RT5 world; `BodiesToRtScene` is a pure function (two calls byte-equal; B bodies → B+1
   prims); `RenderSceneReflected(authScene)` == `RenderSceneReflected(replicaScene)`; the RT render does not mutate
   the `fpx::FxWorld` (pre==post). (If `BodiesToRtScene` lives only in a .cpp/.mm TU, replicate the tiny mapping in
   the test or expose it via a small shared header under `samples`/`tests` — do NOT add a sim dep to `rtrace.h`.)
7. **Introspect.** Add EXACTLY `rt5-simrender` (features) + `--rt5-simrender-shot` (showcases) + update
   `tests/introspect_test.cpp`. CONTROLLER rebakes the JSON golden (verify it stages ` M `).

## RHI seam additions (summary)
- **None.** RT5 reuses the RT2 accel-struct seam + the RT4 `rt_reflect.comp` verbatim. Report the seam: NO
  `rtrace.h` change (BYTE-FROZEN); NO new shader (`rt_reflect.comp` reused); NO sim file change (`fpx.h` READ-ONLY);
  NO `rhi.h`/`vulkan_accel.*`/Metal-backend change; ALL existing shaders + the existing 244 goldens BYTE-UNCHANGED.
  The ONLY new/changed files: the showcase TUs (main.cpp, visual_test.mm), the introspect feature/test, verify.ps1,
  the new test, the new golden, and (optionally) a tiny shared `BodiesToRtScene` header.

## Out of scope (YAGNI)
Rendering the sim DURING the lockstep (RT5 renders the SETTLED end-state — the per-tick render is unnecessary for the
closure). Multiple composed sims (one fpx world). Cloth/fluid/grain → RT (fpx spheres only; other sims' surfaces →
RT is a future opportunistic slice). Rollback rendering (RT5 proves lockstep; the fpx rollback is already its own
proof). A new RT feature (RT5 is the ENVELOPE, not a feature — it reuses RT4's render). RT5 claims ONLY: a
deterministic fpx sim, hardware-ray-traced, is byte-identical between lockstep peers and HW==CPU (the moat↔RT
closure), strict-zero cross-vendor.
