## Design call: the SW reference tracer is the golden; it brute-forces every primitive (no cull)

RT1 establishes the deterministic moat that the whole flagship rests on. The tracer is **pure-integer Q16.16**, so
its result is bit-identical CPU тЖФ Vulkan-GPU тЖФ Metal-CPU. Critically, the closest-hit search **iterates ALL
primitives with no acceleration-structure cull** тАФ this is deliberate: in RT2 the HW inline-ray-query path (which
is AABB-gated by the driver's BVH) will be memcmp'd against THIS no-cull reference, so an insufficient AABB margin
that makes the driver skip a true hit FAILS LOUDLY (the genuine candidate-completeness check, per the roadmap's
crux mitigation). RT1 ships only the reference half of that proof; RT2 adds the HW half.

- **`rtrace::TraceClosest(ray, prims) -> RtHit`** тАФ brute-force over every primitive; keep the running closest by
  the **total order `(t, primIndex)`** (smaller `t` wins; equal `t` тЖТ smaller `primIndex` wins тАФ deterministic
  tie-break). Order-independent тЗТ identical regardless of primitive storage order, vendor, or thread scheduling.
- **All intersection + shade in fx (Q16.16).** `fpx::fxmul` (int64 intermediate, `fpx.h:54`), an `fxdiv` helper
  (`((int64)a << kFrac) / b`, integer division тАФ deterministic, identical DXCтЖФCPU) for the AABB slab test, and
  `fpx::FxISqrt` (`fpx.h:78`) for the sphere discriminant root and the shade-normal normalize. NEVER any float,
  `std::sqrt`, or `<cmath>` on the bit-exact path. Degenerate rays (`dir == 0`) rejected at scene-build.
- **The render is INTEGER-shaded** (`ShadeHitInt`: clamped fx Lambert `max(dot(n,L),0)` ├Ч a `primIndex`-tinted
  albedo + small ambient, quantized to RGBA8; miss тЖТ fixed background) тЖТ the `rt1_trace` golden is **strict-zero
  cross-vendor** (RT1тАУRT5 are the strict tier; the float visresolve money-shot is reserved for RT6). On Vulkan the
  showcase writes the **GPU**-produced image (and proves GPU==CPU), so it equals the Metal (CPU) image by
  construction.

> NOTE: one thread per pixel, inner loop over a handful of analytic primitives тАФ a single compute dispatch is FAR
> under the Windows ~2 s TDR watchdog (see [[hazard-forge-gpu-tdr-chunking]]); **no chunking needed**. Keep the
> scene small (тЙИ4тАУ8 primitives: a few spheres + AABB boxes + a large ground AABB).

## Design decisions (locked)
1. **NEW `engine/render/rtrace.h`** (`hf::render::rtrace`), the flagship's append-only header (frozen after RT1 for
   RT2+ to `#include` read-only). Contents:
   - `struct RtSphere { math::FxVec3 center; fx radius; uint32_t primIndex; };`
     `struct RtAabb { math::FxVec3 lo, hi; uint32_t primIndex; };` (Q16.16 only). A small `RtScene` holding spans
     of each (SoA), plus a fixed directional `lightDir` (fx) and background color.
   - `struct RtRay { math::FxVec3 origin, dir; };` (fx; `dir` NOT required unit-length тАФ `t` is in units of
     `|dir|`, consistent within a ray so the per-ray min is valid; build rejects `dir==0`).
   - `struct RtHit { fx t; uint32_t primIndex; math::FxVec3 pos, normal; };` + `kRtMiss` sentinel
     (`primIndex = 0xFFFFFFFFu`, `t = kRtNoHit` = INT32_MAX).
   - `IntersectSphere(ray, sphere, RtHit&)` (fx quadratic; discriminant тЙе0 via `FxISqrt`; nearest non-negative
     root), `IntersectAabb(ray, aabb, RtHit&)` (fx slab test via `fxdiv`; entry-`t`, face normal), each returning
     whether it hit and writing `t`/`pos`/`normal`.
   - `TraceClosest(ray, scene) -> RtHit` (brute-force ALL prims, `(t,primIndex)` total-order min, NO cull).
   - `ShadeHitInt(hit, scene) -> uint32_t` (integer Lambert тЖТ RGBA8; miss тЖТ background).
   - `RenderScene(scene, camera, width, height, std::span<uint32_t> outRGBA8)` тАФ integer primary-ray gen from an
     fx camera basis (NO float), trace, shade. Deterministic, pure CPU.
   - A `BuildRt1Scene()` helper returning the fixed showcase scene + camera.
2. **NEW `shaders/rt_trace.comp.hlsl`** тАФ one thread per pixel; the SAME `TraceClosest`/`ShadeHitInt`/ray-gen math
   in HLSL (int64 fx mul/div + the `FxISqrt` digit loop, mirroring `mc.h::ISqrt`); writes RGBA8 (or `RtHit`) to an
   SSBO. **DXC `:cs` line in `samples/hello_triangle/CMakeLists.txt` ONLY; NO `hf_gen_msl` line** (Vulkan-SPIR-V-
   only; Metal runs the CPU reference тАФ the `fpx_integrate.comp` convention; add the SAME explanatory comment block
   in `metal_headless/CMakeLists.txt`). NO new pipeline type тАФ inline compute (the existing `ComputePipelineDesc` /
   `BindStorageBuffer` / `DispatchCompute` / readback path).
3. **Accel-struct RHI seam тАФ additive to `engine/rhi/rhi.h`, defaulted-no-op, INERT in RT1.** This flagship is the
   one place the "rhi.h is frozen" rule is deliberately relaxed: **additive only**, existing signatures
   byte-unchanged, existing goldens byte-unchanged, backends NOT modified (they inherit the defaults). Add:
   - `class IAccelStructure { public: virtual ~IAccelStructure() = default; };` (opaque handle, by `IBindlessTextureSet`).
   - `struct AccelGeometry { enum class Kind { AabbProcedural, Triangle } kind; /* AABB: */ math::Vec3 lo, hi;
     /* Triangle: */ const float* verts; uint32_t vertCount; uint32_t firstPrim; };` (RT1 DEFINES; RT2 USES).
   - `struct BlasDesc { std::span<const AccelGeometry> geoms; };`
     `struct TlasInstance { IAccelStructure* blas; float transform[12]; uint32_t instanceId; };`
     `struct TlasDesc { std::span<const TlasInstance> instances; };`
   - On `IRHIDevice`: `virtual std::unique_ptr<IAccelStructure> CreateBlas(const BlasDesc&) { return nullptr; }`,
     `virtual std::unique_ptr<IAccelStructure> CreateTlas(const TlasDesc&) { return nullptr; }`,
     `virtual bool SupportsHardwareRayQuery() const { return false; }`.
   - On `ICommandBuffer`: `virtual void BindAccelStructure(IAccelStructure& /*tlas*/, uint32_t /*slot*/) {}`.
   - RT1 does NOT override these anywhere. The SW tracer does NOT use them. They exist solely to land the new RHI
     inert + prove the existing goldens are unaffected, de-risking RT2.
4. **Showcase `--rt1-trace-shot <out>` (Vulkan) AND `--rt1-trace` (Metal) тАФ WIRE BOTH (grep your OWN
   `metal_headless/visual_test.mm` for `--rt1-trace` BEFORE reporting DONE; see
   [[hazard-forge-metal-showcase-gate]]).** Both build `BuildRt1Scene()`, render the image, write it.
   - **Vulkan:** dispatch `rt_trace.comp` тЖТ read back the RGBA8 SSBO; ALSO compute the CPU `rtrace::RenderScene`
     reference; **memcmp strict**; write the GPU image to `<out>`. Emit the proofs.
   - **Metal:** run the CPU `rtrace::RenderScene`, write the image. Golden = `tests/golden/metal/rt1_trace.png`
     (Mac-baked by the CONTROLLER тАФ DO NOT commit).
5. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) GPU==CPU (Vulkan):** `rt1-trace: {rays:<R>, prims:<P>, hits:<H>} GPU==CPU image BYTE-EQUAL` тАФ
     `std::memcmp` of the GPU readback vs the CPU `RenderScene` output == 0.
   - **(2) analytic correctness:** `rt1-trace: analytic OK (sphere t=<...>, aabb t=<...>, miss ok)` тАФ a ray fired
     at a sphere center hits at `t == dist - radius` (closed form); a ray fired into a known AABB face hits at the
     slab entry `t`; a ray pointed at empty space returns `kRtMiss`. Pins CORRECTNESS, not just determinism.
   - **(3) determinism:** CPU `RenderScene` two-run byte-identical; the Vulkan path exits 0 + writes the image;
     Metal two-run DIFF 0.0000 (controller).
   - **(4) seam inert:** `rt1-trace: rhi accel seam present (hwRayQuery:<bool>, blas:default-null)` тАФ call
     `dev.SupportsHardwareRayQuery()` (returns the real capability or the default false) and confirm `CreateBlas`

   - Golden discipline: ONLY `tests/golden/metal/rt1_trace.png`; do NOT commit it. Existing 239 goldens UNTOUCHED.
6. **Cross-backend bar:** the COMMITTED golden is the Mac-Metal bake; `verify.ps1` re-renders + compares at

7. **Tests тАФ NEW `tests/rt_test.cpp` (pure CPU, ASan-clean), registered in `tests/CMakeLists.txt` (ctest

8. **Introspect.** Add EXACTLY `rt1-trace` (features) + `--rt1-trace-shot` (showcases) + update

## RHI seam additions (summary)
- **The accel-struct seam IS the new RHI** (additive, defaulted-no-op, inert in RT1): `IAccelStructure`,
  `AccelGeometry`/`BlasDesc`/`TlasInstance`/`TlasDesc`, `IRHIDevice::{CreateBlas,CreateTlas,SupportsHardwareRayQuery}`,
  `ICommandBuffer::BindAccelStructure` тАФ ALL with default bodies; NO backend override in RT1. Report the seam:
  `rhi.h` ADDITIVE-only (existing decls byte-unchanged), NO Vulkan/Metal backend change, the SW tracer + GPU kernel
  use ONLY the existing compute path. `engine/render/rtrace.h` is NEW. `shaders/rt_trace.comp.hlsl` is NEW
  (Vulkan-only). NO sim header / sim shader / sim golden touched (RT is render-only). NO existing shader changed.

## Out of scope (YAGNI)
HW inline ray query + ACTUALLY building/binding accel structures (RT2 тАФ RT1 only DEFINES the seam). Triangle-mesh
geometry & barycentric/float radiance (Tier B / RT6). RT shadows / AO (RT3). RT reflections (RT4). The
determinism-envelope + lockstep tie-in (RT5). The float visresolve money-shot capstone (RT6). A BVH or any spatial
acceleration in the SW tracer (brute-force is DELIBERATE тАФ it is the completeness oracle for RT2). RT1 claims ONLY:
a deterministic Q16.16 SW reference ray tracer whose GPU twin is bit-identical (GPU==CPU memcmp on Vulkan; Metal CPU
== Vulkan GPU by construction), rendering a strict-zero-cross-vendor `rt1_trace` golden, PLUS an inert additive
accel-struct RHI seam that leaves all 239 existing goldens byte-unchanged. This is the FIRST slice тЖТ FLAGSHIP #28
BEGINS.
