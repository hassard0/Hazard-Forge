**Goal:** Append to `engine/game/verdict.h`: a deterministic snapshot **digest**, a serializable **replay file**, and
record/verify entry points:
- `std::string DigestSnapshot(const VerdictSnapshot& s)` — a 16-hex FNV-1a over a CANONICAL byte-encoding of the whole
  heterogeneous snapshot (tick, nextId, order[], every component pool in order[] sequence, the sim TRIPLE) — a compact,
  deterministic, backend-agnostic fingerprint of the final world.
- `struct ReplayFile { uint32_t ticks; std::vector<Command> stream; std::string finalDigest; }` (the canonical scene
  is a FIXED builder both record + verify call, so the initial world need not be serialized — see the design call).
- `std::string SerializeReplay(const ReplayFile&)` — deterministic JSON text `{ticks, stream:[{tick,kind,target,arg}],
  finalDigest}` (the house `%g`/integer style; arg is the Q16.16 `FxVec3` as integers).
- `ReplayFile ParseReplay(const std::string& text)` — the inverse (the vendored JSON parser).
- `bool VerifyReplay(const ReplayFile&)` — rebuild the canonical world, `RunVerdictLockstep` over `stream` for `ticks`,
  and return `DigestSnapshot(final) == finalDigest`.

## Design call: a fixed canonical scene + a digest of the final world; the replay is {stream, ticks, digest}
The replay must be self-contained and reproducible. Rather than serialize the entire initial world, DX5 fixes the
INITIAL state as a canonical scene builder both `--replay-record` and `--replay-verify` (and the test) call:
- **`BuildCanonicalReplay() -> (VerdictWorld world0, VerdictParams params, std::vector<Command> stream, uint32_t
  ticks)`** — the FIXED canonical replay scene (a VD3/VD5-style composed gameplay+physics world + a fixed command
  stream that nudges a body, spawns/collects, etc.). Record and verify build the IDENTICAL world0 from this, so the
  replay file only needs `{stream, ticks, finalDigest}` — the initial state is implied by the canonical builder. (This
  keeps DX5 self-contained; serializing arbitrary initial worlds is a future extension, out of scope.)
- **The digest is over the FINAL world** (`SnapshotWorld(RunVerdictLockstep(...))`), the moat's payload: two machines
  (or two runs) that replay the same stream produce the same final world ⇒ the same digest. `DigestSnapshot` walks the
  snapshot in a FIXED canonical order (the `order[]`-keyed sequence the determinism contract already uses) and FNV-1a's
  it — so the digest is a pure function of the deterministic final state, backend-identical.
- **The golden `canonical.replay`** is the serialized `{stream, ticks, finalDigest}` — a deterministic text artifact,
  byte-compared. It is BOTH the record output AND the verify input (round-trip).
- **Tamper-detect** (the falsifiable beat): flip ONE command's `arg` (or `kind`) in the parsed stream → re-run →
  `DigestSnapshot(final)` DIFFERS from `finalDigest` (`VerifyReplay` returns false). The replay catches drift — a
  changed input or a changed engine both move the digest, which is exactly the regression-guard value.

## Design decisions (locked)
1. **APPEND to `engine/game/verdict.h`** (`hf::game::verdict`, VD1–VD6 byte-frozen — do NOT edit existing lines):
   `DigestSnapshot`, `ReplayFile`, `SerializeReplay`, `ParseReplay`, `BuildCanonicalReplay`, `VerifyReplay`. All pure
   CPU. `DigestSnapshot` MUST walk the snapshot in the `order[]`-keyed canonical sequence (the determinism contract) so
   it's stable under ECS handle churn.

   `RunVerdictLockstep` → `DigestSnapshot(final)` → `SerializeReplay({stream,ticks,digest})` → write `<out.replay>`
   LF-clean (no BOM). **`--replay-verify <in.replay>`**: `ParseReplay` → `VerifyReplay` → assert true (+ the
   lockstep authority==replica proof). Wire the Metal `--replay-record`/`--replay-verify` in `metal_headless/visual_test.mm`
   for symmetry. **Grep your OWN visual_test.mm for `--replay-record` before reporting DONE.** (MSVC C1061 → the DX1-4
   pre-scan/fold workaround.)
3. **Register `--replay-record` + `--replay-verify` in `kShowcases`**. Churns `agentApi.contentHash`; `schemaVersion`
   STAYS 1. Note before/after `contentHash`. (The `replay` capability — enumerated since DX1 — is now realized.)
4. **PROOFS (exact stdout lines, fail loudly):**
   - `dx5-replay record: {ticks:<K>, commands:<C>, digest:<hex>} -> replay file byte-golden MATCH`
   - `dx5-replay verify: replayed world digest == recorded digest BIT-IDENTICAL`
   - `dx5-replay lockstep: RunVerdictLockstep(stream) authority==replica (whole world)`

harness with a golden replay file and a tamper-detect guard, productizing the lockstep moat as a regression artifact.
