## Design call: a shadow ray is an any-hit occlusion test — order-independent, hence deterministic
After the primary `TraceClosest` finds the surface point, cast a shadow ray from `hit.pos + normal*kRtShadowEps`
(the offset kills self-shadow acne) toward `scene.lightDir`. The point is in shadow iff ANY primitive is intersected
with `t > kRtShadowMinT` (directional light → no far bound). "Any hit" is a boolean OR over OUR fx intersections of
the candidates — **independent of traversal order**, so the HW any-hit (drain candidates, OR our fx hits) equals the
SW brute-force any-hit equals the CPU, bit-exact. The shade gates the diffuse term: `ambient + (occluded ? 0 :
diffuse)`, so shadowed pixels keep only ambient (visibly darker) — the integer shadowed image is strict-zero
cross-vendor.
- **The shadow ray CAN early-out** (the moment one occluder is found) WITHOUT breaking determinism, because the
  result is a boolean (presence of an occluder), not an order-dependent min. The HW path may early-out on its first
  candidate hit; the SW reference may scan all — both yield the same boolean. (Unlike the primary ray, which MUST
  drain for the closest-hit min.)
- **Same three constraints as RT2** (the AABB margin makes the float traversal a superset; all intersection in fx;
  degenerate rays rejected) carry over verbatim — the shadow ray reuses the inflated-AABB accel structure.

## Design decisions (locked)
1. **APPEND to `engine/render/rtrace.h`** (RT1 byte-frozen):
   - `inline constexpr fx kRtShadowEps  = kOne / 256;`  (surface offset along the normal)
     `inline constexpr fx kRtShadowMinT = kOne / 1024;` (ignore hits closer than this — self-hit guard)
   - `bool TraceAnyHit(const RtRay& ray, const RtScene& scene, fx minT)` — brute-force; return true on the FIRST
     primitive whose `IntersectSphere`/`IntersectAabb` yields `t > minT` (order-independent boolean). NO closest-hit.
   - `uint32_t ShadeHitShadowed(const RtHit& hit, const RtScene& scene, bool occluded)` — `if miss → background`;
     else `diffuse = ambient + (occluded ? 0 : fxmul(kOne-ambient, max(dot(n,L),0)))`, albedo-tinted, quantized
     (the `ShadeHitInt` body with the diffuse gated). When NOT occluded this is byte-identical to `ShadeHitInt`.
   - `uint32_t RenderSceneShadowed(scene, cam, w, h, out)` — per pixel: `TraceClosest`; if hit, build the shadow ray
     `{hit.pos + normal*kRtShadowEps, lightDir}` and `occluded = TraceAnyHit(shadowRay, scene, kRtShadowMinT)`;
     `out = ShadeHitShadowed(hit, scene, occluded)`. Returns the count of SHADOWED pixels (for the proof).
2. **NEW `shaders/rt_shadow.comp.hlsl`** (Vulkan-SPIR-V-ONLY, int64 + RayQuery, `csrq` stage): one thread per pixel.
   Primary `RayQuery` drain → closest hit (the `rt_query.comp` body). If hit: compute the fx shadow-ray origin/dir,
   open a SECOND `RayQuery` over `gTlas`, drain (or early-out) the bounding-box candidates running OUR fx
   `IntersectSphere`/`IntersectAabb`, set `occluded` if any yields `t > kRtShadowMinT`. Write `ShadeHitShadowed`.
   DXC `csrq` line in `samples/hello_triangle/CMakeLists.txt` ONLY; the Vulkan-only comment block in
   `metal_headless/CMakeLists.txt`; NO `hf_gen_msl` line.
3. **Showcase `--rt3-shadow-shot <out>` (Vulkan) AND `--rt3-shadow` (Metal) — WIRE BOTH (grep your OWN
   `visual_test.mm` for `--rt3-shadow` BEFORE reporting DONE; [[hazard-forge-metal-showcase-gate]]).** The RT2 scene
   (ground AABB + 4×4 sphere grid + two boxes — the spheres/boxes cast shadows on the ground under the directional
   light). Vulkan (HW present): build the accel structure (reuse the `--rt2-query-shot` BLAS/TLAS build), dispatch
   `rt_shadow.comp`, read back; compute CPU `RenderSceneShadowed`; memcmp HW==CPU strict; write the HW image. No-RT
   device → CPU fallback + the `hwRayQuery:false → SW` line. Metal: CPU `RenderSceneShadowed`. Golden =
   `tests/golden/metal/rt3_shadow.png` (Mac-baked by the CONTROLLER).
4. **PROOFS (fail loudly; exact stdout lines):**
   - **(1) HW==CPU:** `rt3-shadow: {rays:<R>, prims:<P>, shadowed:<S>} HW==CPU image BYTE-EQUAL (hwRayQuery:true)`.
   - **(2) shadows exist:** `rt3-shadow: shadows present (shadowed:<S> in (0, lit))` — `0 < shadowedCount <
     primaryHitCount` (the shadow ray actually darkened some lit surface — not all-lit, not all-dark; a real
     occlusion sanity, not just determinism).
   - **(3) determinism:** CPU `RenderSceneShadowed` two-run byte-identical; HW two-run byte-identical.
   - **(4) capability:** `hwRayQuery:true` on the RTX 4070 (the no-HW fallback prints `hwRayQuery:false → SW`).
   - Golden discipline: ONLY `tests/golden/metal/rt3_shadow.png`; do NOT commit it. Existing 242 goldens UNTOUCHED.
5. **Cross-backend bar:** strict **0.0000** (integer shade). Mac re-renders `--rt3-shadow` (CPU ref) vs the committed
   golden at 0.0000; the controller records the Vulkan-HW vs Metal cross-vendor diff (expect 0.0000).
6. **Tests — NEW `tests/rt_shadow_test.cpp` (pure CPU, ASan-clean), registered in `tests/CMakeLists.txt` (ctest

## Out of scope (YAGNI)
Soft shadows / penumbra (hard shadows only — a single shadow ray per pixel toward the directional light). RT AO
(hemisphere sampling — a possible RT3b/RT4 companion; deferred to keep the slice bounded + the proof clean). RT
reflections (RT4). Metal HARDWARE shadows (RT3's Metal runs the CPU reference; the Metal-HW shadow parity is a
follow-on — RT2b proved the Metal HW primary path). Multi-light shadows (one directional light). Triangle geometry /
float radiance (Tier B / RT6). RT3 claims ONLY: deterministic HARD shadows via a second any-hit ray query, byte-
identical HW==CPU on Vulkan HW, strict-zero cross-vendor, with a real-occlusion sanity.
